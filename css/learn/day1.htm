Day 1: CSS 基礎與選擇器

CSS Basic and Selectors

一、學習目標

完成本日課程後，你將能夠：

  • 理解 CSS 的作用與基本概念
  • 掌握 CSS 的引入方式並選擇適合的方法
  • 使用元素選擇器、Class 選擇器、ID 選擇器
  • 為連結套用不同狀態的樣式（ :link, :visited, :hover, :active ）
  • 建立第一個包含外部 CSS 的網頁

二、什麼是 CSS？

CSS (Cascading Style Sheets) 是層疊樣式表，用來控制網頁元素的外觀與排版。如果說 HTML 是網頁的骨架，那 CSS 就是皮膚和衣服，負責讓網頁變得美觀。

網頁的三大核心：

  • HTML = 網頁的結構與內容（骨架）
  • CSS = 網頁的外觀與樣式（皮膚和衣服）
  • JavaScript = 網頁的互動與行為（大腦和肌肉）

Cascading 層疊的意義： 當多個 CSS 規則應用在同一個元素上時，會依照「優先級」和「順序」來決定最終樣式。後面的規則會覆蓋前面的規則。

三、CSS 基本語法結構

CSS 由「選擇器」和「宣告區塊」組成：

  • 選擇器 (Selector)：指定要套用樣式的 HTML 元素
  • 屬性 (Property)：要改變的樣式特性
  • 值 (Value)：屬性的具體設定

CSS 的三種引入方式

  • Inline Style 內聯樣式
  • Internal Style 內部樣式
  • External Style 外部樣式

1. Inline Style 內聯樣式

直接寫在 HTML 標籤的 style 屬性中。


<p style="color: red;">紅色文字</p>
  • 優點：優先級最高，可快速測試單一元素樣式
  • 缺點：難以維護、無法重複使用、違反關注點分離原則 註
  • 使用時機：僅在特殊情況、緊急修正，或 JavaScript 動態生成樣式時使用

備注：關注點分離原則（Separation of Concerns, SoC）旨在將網頁結構、樣式與行為拆分（HTML 負責結構，CSS 負責樣式），以提升程式碼的可讀性、可維護性與重用性，並利於團隊分工。

2. Internal Style 內部樣式

寫在 HTML 的 <head> 區塊內的 <style> 標籤中。


<!DOCTYPE html>
<html lang="zh-TW">
<head>
    <meta charset="UTF-8">
    <title>Internal Style 內部樣式</title>
    <style>
        p {color: red;}
    </style>
</head>
<body>
    <p>這是一段紅色的文字</p>
</body>
</html>
  • 優點：適合單頁面的樣式、無需加載額外檔案
  • 缺點：無法跨頁面共用、增加 HTML 檔案複雜性、不利於快取
  • 使用時機：小型專案、單頁應用，或特定頁面專屬樣式

3. External Style 外部樣式

建立獨立的 .css 檔案並引入，有兩種引入方法：

方法 A: 使用 HTML 的 <link> 標籤（推薦）


<!DOCTYPE html>
<html lang="zh-TW">
<head>
    <meta charset="UTF-8">
    <title>External Style 外部樣式</title>
    <link rel="stylesheet" href="style.css">
</head>
<body>
    ...
</body>
</html>
  • 優點：可重複使用、易於維護、瀏覽器可平行下載、支援媒體查詢、可被瀏覽器快取，提升效能
  • 缺點：需要額外的 HTTP 請求
  • 使用時機：這是最推薦的方式，適用於所有專案

方法 B: 使用 CSS 的 @import 規則


<!DOCTYPE html>
<html lang="zh-TW">
<head>
    <meta charset="UTF-8">
    <title>External Style 外部樣式</title>
    <style>
        @import url('style.css');
    </style>
</head>
<body>
    ...
</body>
</html>
  • 優點：可以在 CSS 內部管理多個樣式表、適合模組化組織
  • 缺點：效能較差（會阻塞渲染）、無法平行下載
  • 使用時機：適合開發階段模組化管理，生產環境建議打包成單一檔案

特別注意事項：在 CSS 檔中使用 @import 規則時，@import 必須置於 CSS 內容的最上方。

<link> 跟 @import 比較表

特性 <link> @import
位置 HTML <head> CSS 檔案內或 <style> 標籤
載入方式 平行下載 ⚡ 依序下載 🐌
效能 較好 較差
推薦程度 ⭐⭐⭐⭐⭐ ⭐⭐

<link> 跟 @import 雖然同為 External Style ，但實際使用上還有許多細微的差異。若想要進一步瞭解這些差別，請詳見 <link> 跟 @import 比較說明。

四、CSS 的選擇器

「CSS 選擇器」—— 顧名思義，就是用來選擇要設定 CSS 的工具。比較常見的 CSS 選擇器有四種：

  • Element Selector 元素選擇器
  • Class Selector 類別選擇器
  • ID Selector ID 選擇器
  • Pseudo-classes 偽類選擇器

1. Element Selector 元素選擇器

直接使用 HTML 標籤名稱作為選擇器。


/* 選取頁面中所有的 <p> 元素 */
p {
  color: #333;
}
/* 選取頁面中所有的 <h1> 元素 */
h1 {
  font-size: 32px;
}

特性：

  • 會影響頁面上所有該類型的元素
  • 最基本、最簡單的選擇器

使用時機：

  • 設定整體網頁的基礎樣式
  • 重置或標準化元素的預設樣式

2. Class Selector 類別選擇器

使用 . 開頭，搭配 HTML 的 class 屬性作為選擇器。


/* 類別選擇器 */
.highlight {
  background-color: yellow;
}

.text-center {
  text-align: center;
}



<p>這是普通文字</p>
<p class="highlight">這是重點文字</p>
<p class="hilight text-center">這是重點又居中的文字</p>

特性：

  • 可重複使用在多個元素上
  • 一個元素可以有多個 class（用空格分隔）
  • 優先級比元素選擇器高

命名建議：

  • 使用小寫英文字母
  • 多個單字用 - 連接（kebab-case）
  • 語意化命名（描述用途而非外觀）
    • ✅ .btn-primary, .card-header
    • ❌ .red-button, .big-text

使用時機：

  • 需要重複使用的樣式
  • 建立元件或模組
  • 最常用的選擇器

3. ID Selector ID 選擇器

使用 # 開頭，搭配 HTML 的 id 屬性。


/* ID 選擇器 */
#main-header {
  background-color: navy;
  color: white;
}

#main-footer {
  margin-top: 50px;
}



<header id="main-header">
  <h1>網站標題</h1>
</header>

<footer id="main-footer">
  <p>版權資訊</p>
</footer>

特性：

  • 在一個頁面中應該是唯一的（只能出現一次）
  • 優先級比 class 高
  • 也用於 JavaScript 選取元素和頁面錨點

使用時機：

  • 選取頁面上唯一的元素（如 header、footer）
  • JavaScript 操作特定元素
  • 頁面內錨點連結

注意：

  • 現代開發中，CSS 樣式通常優先使用 class 而非 ID
  • ID 的高優先級可能造成樣式難以被覆蓋

4. Pseudo-classes 偽類選擇器

Pseudo （唸起來像速度）偽類選擇器，通常用在選取 HTML 元素的特定「狀態」。偽類選擇器相當多，為了降低學習複雜度，這邊先介紹 <a> 連結的四個重要狀態偽類。


/* 必須按照 LVHA 順序 */

/* :link - 未訪問的連結 */
a:link {
  color: #3498db;
  text-decoration: none;
}

/* :visited - 已訪問的連結 */
a:visited {
  color: #9b59b6;
}

/* :hover - 滑鼠懸停 */
a:hover {
  color: #e74c3c;
  text-decoration: underline;
}

/* :active - 點擊當下 */
a:active {
  color: #c0392b;
}

LVHA 順序規則：

超連結的偽類的順序非常重要，必須按照 L-V-H-A 的順序撰寫：

  • :link → :visited → :hover → :active

為什麼順序重要？ 因為 CSS 的「層疊」特性，後面的規則會覆蓋前面的規則。如果順序錯誤，某些狀態可能無法正常顯示。

使用時機：

  • 為連結提供視覺回饋
  • 改善使用者體驗
  • 區分已訪問和未訪問的連結

選擇器優先級基礎

當多個選擇器選取同一個元素時，優先級決定哪個樣式會生效。

優先級順序（從高到低）：

  • Internal Style 內部樣式 (HTML 元素的 style 屬性)
  • ID 選擇器 (#id)
  • Class 選擇器 (.class)
  • 元素選擇器 (HTML 元素)

當優先級相同時：

  • 讀取順序排在後面的規則會覆蓋掉前面的規則

測試範例：


p {
  color: black;
}

.highlight {
  color: blue;
}

#intro {
  color: red;
}



<!-- 這段語法同時符合了三種選擇器的設定  -->
<p id="intro" class="highlight">這是一段測試文字</p>

結果： 文字顯示為紅色，因為 ID 選擇器優先級最高。

以上為最常見的幾組基本 CSS 選擇器，初學者只要先學會上述幾種就可以開始實作。若想要進一步瞭解選擇器的功能，請詳見更完整的 CSS 選擇器介紹。

五、常見錯誤與注意事項

錯誤 1: CSS 語法錯誤

錯誤示範：


/* ❌ 忘記分號：等於 color 這一行設定沒有結束 */
p {
  color: red /* 應該要加上 ; 收尾 */
  font-size: 16px;
}

/* ❌ 忘記冒號：直接是無效的 CSS 設定 */
p {
  color red;
}

/* ❌ 忘記大括號：直接是無效的 CSS 設定 */
p
  color: red;
  font-size: 16px;

/* ❌ 屬性英文名稱拼錯：直接是無效的 CSS 設定 */
p {
  collor: red;  /* 應該是 color */
}

正確做法：


/* ✅ 正確語法 */
p {
  color: red;
  font-size: 16px;
}

注意事項：

1. CSS 不區分大小寫（屬性和值），但建議全部用小寫


/* 技術上都有效，但不推薦混用 */
   p { COLOR: RED; }
   p { Color: Red; }
   /* ✅ 推薦：全部小寫 */
   p { color: red; }

2. Class 和 ID 名稱區分大小寫


<div class="Card"></div>



.card { }   /* ❌ 第一個字小寫，無法選取 */
.Card { }   /* ✅ 大小寫完全符合，可以選取 */

3. 可以使用開發者工具來檢查樣式

  • 按 F12 開啟瀏覽器開發者工具
  • 可以即時查看和修改 CSS
  • 可以看到哪些樣式被套用、哪些被覆蓋

錯誤 2: 忘記在 class 和 ID 前加符號

錯誤示範：


/* ❌ 錯誤：這會選取叫做 <highlight> 的自訂 HTML 元素，而不是 class */
highlight {
  background-color: yellow;
}

/* ❌ 錯誤：這會選取 <header> HTML 元素，而不是 ID */
header {
  background-color: navy;
}

正確做法：


/* ✅ 正確：class 選擇器要加 . */
.highlight {
  background-color: yellow;
}

/* ✅ 正確：ID 選擇器要加 # */
#header {
  background-color: navy;
}

錯誤 3: CSS 檔案路徑錯誤

錯誤示範：


**檔案結構：**

project/
├── index.html
├── css/
│   └── style.css
└── images/

<!-- ❌ 錯誤：檔案不在正確位置 -->
<link rel="stylesheet" href="style.css">
<!-- 實際檔案在 css/style.css -->

正確做法：


<!-- ✅ 正確：路徑要對應實際檔案位置 -->
<link rel="stylesheet" href="css/style.css">

六、重點整理

  • CSS 的作用：控制網頁元素的外觀與樣式/li>
  • 引入方式：外部樣式（使用 <link>）是最推薦的方式
  • 元素選擇器：直接使用標籤名稱，影響所有該類型元素
  • Class 選擇器：使用 . 開頭，可重複使用，最常用
  • ID 選擇器：使用 # 開頭，應該是唯一的
  • 連結偽類：必須按照 LVHA 順序（:link → :visited → :hover → :active）
  • 優先級：內聯 > ID > Class > 元素
  • 語法結構：選擇器 { 屬性: 值; }
