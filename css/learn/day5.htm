Day 5: Display 與 Position 定位

CSS Display and Position

本單元算是 「CSS 網頁排版」相關的第一個章節，尤其是 CSS Position 部分，雖然是相當早期的 CSS 技術，但是因為在排版上相當實用，至今還是非常重要常見的排版技術。

一、學習目標

完成本日課程後，你將能夠：

  • 理解 display 屬性的作用與常用值
  • 掌握 block、inline、inline-block 的差異
  • 了解 position 屬性的五種定位方式
  • 使用 top、right、bottom、left 控制元素位置
  • 理解 z-index 的層疊順序控制

二、Display 屬性

Display 屬性決定元素如何顯示及與其他元素的排列方式，有許多種不同的性設定：

  • display: block （顯示為區塊元素）
  • display: inline （顯示為行內元素）
  • display: inline-block （顯示為行內區塊）
  • display: none （不顯示）
  • display: flex （會在後面的章節介紹）
  • display: grid （會在後面的章節介紹）

1. block 區塊元素

HTML 元素有許多預設的屬性，block 區塊元素就是其中之一。block 元素通常用來作為網頁內容的「分類」與「骨架結構」，是排版專用的 HTML 元素。


/* Block - 區塊元素 */
.block {
  display: block;
  width: 200px;
  height: 100px;
  background-color: #3498db;
  margin: 10px 0;
}

特性：

  • 獨佔一行空間，後面的元素會自動換行
  • 可以設定寬度和高度
  • 可以設定 Margin 和 Padding
  • 預設寬度為父元素的 100%

預設為 block 的元素： <div>, <p>, <h1>-<h6>, <ul>, <ol>, <li>, <section>, <header>, <footer>⋯等。

注意：網頁中的圖片 <img> 雖然方方正正的，看起來像是個 Block ，但其實不是 block 元素，而是屬於 inline 元素。

2. inline 行內元素

行內元素元素也是 HTML 元素許多預設的屬性之一。


/* Inline - 行內元素 */
.inline {
  display: inline;
  background-color: #e74c3c;
  /* width 和 height 無效 */
}

特性：

  • 不會換行，與其他元素在同一行
  • 不能設定寬度和高度
  • 寬度由內容決定
  • 垂直方向的 margin 和 padding 無效

預設為 inline 的元素： <span>, <a>, <strong>, <em>, <img>, <input>⋯等。

3. inline-block 行內區塊


/* Inline-block - 行內區塊 */
.inline-block {
  display: inline-block;
  width: 100px;
  height: 50px;
  background-color: #2ecc71;
  margin: 0 10px;
}

特性：

  • 不會換行，與其他元素在同一行
  • 可以設定寬度和高度
  • 結合了 inline 和 block 的優點

特性：

  • 導覽列按鈕
  • 圖片排列
  • 標籤或徽章

4. none 隱藏元素

用來讓指定 HTML 元素從網頁中完全消失。


/* None - 隱藏 */
.hidden {
  display: none;
}

特性：

  • 元素完全不顯示
  • 不佔據空間

與 visibility: hidden 的差異：

  • display: none - 不顯示，不佔空間
  • visibility: hidden - 不顯示，但仍佔空間

Display 屬性比較表

屬性 block inline inline-block
換行 是 否 否
設定寬高 可以 不可以 可以
垂直 margin/padding 有效 無效 有效
預設寬度 100% 內容決定 內容決定

三、Position 屬性

position 屬性決定元素的位置，一共有五種定位設定方式：

  • position: static（靜態定位，預設值）
  • position: relative（相對定位）
  • position: absolute（絕對定位）
  • position: fixed（固定定位）
  • position: sticky（黏著定位）

1. static（靜態定位，預設值）

特性：

  • 元素按照正常文檔流（從上到下、由左而右）排列
  • top、right、bottom、left、z-index 無效

使用場景：

  • 當作橡皮擦，用來清除先前設定的定位屬性

2. relative（相對定位）


/* Relative - 相對定位 */
.relative {
  position: relative;
  top: 20px;    /* 向下移動 20px */
  left: 30px;   /* 向右移動 30px */
  /* 原本的空間仍然保留 */
}

特性：

  • 相對於元素原本的位置進行偏移
  • 原本所在的空間仍然保留
  • 不影響其他元素的位置

使用場景：

  • 微調元素位置
  • 作為 absolute 定位的參考點（父元素）

3. absolute（絕對定位）


/* Absolute - 絕對定位 */
.parent {
  position: relative;  /* 作為參考點 */
  width: 300px;
  height: 200px;
  background-color: #ecf0f1;
}

.child {
  position: absolute;
  top: 20px;
  right: 20px;
  width: 100px;
  height: 50px;
  background-color: #e74c3c;
}

特性：

  • 相對於最近的 已定位父元素（非 static）進行定位
  • 如果沒有已定位的父元素，則相對於 <body> 標籤位置（也就是網頁頁面的左上角）
  • 設定絕對定位的元素會脫離正常文檔流，不佔空間
  • 其他元素會忽略它的存在

使用場景：

  • 彈出視窗
  • 下拉選單
  • 浮動按鈕
  • 徽章或標記

4. fixed（固定定位）


/* Fixed - 固定定位 */
.fixed-header {
  position: fixed;
  top: 0;
  left: 0;
  width: 100%;
  background-color: #2c3e50;
  color: white;
  padding: 20px;
  z-index: 1000;
}

.back-to-top {
  position: fixed;
  bottom: 20px;
  right: 20px;
  width: 50px;
  height: 50px;
  background-color: #3498db;
  border-radius: 50%;
}

特性：

  • 相對於瀏覽器視窗進行定位
  • 脫離正常文檔流
  • 捲動頁面時位置不變

使用場景：

  • 固定導覽列
  • 回到頂部按鈕
  • 浮動客服視窗

5. sticky（黏著定位）


/* Sticky - 黏性定位 */
.sticky-nav {
  position: sticky;
  top: 0;  /* 捲動到頂部時固定 */
  background-color: white;
  border-bottom: 2px solid #ddd;
  padding: 10px;
}

特性：

  • 結合 relative 和
  • 在捲動到指定位置前表現為 relative
  • 捲動到指定位置後表現為 fixed
  • 需要設定 top、bottom 等值

使用場景：

  • 頁首選單
  • 表格標題
  • 側邊欄
  • 區塊標題

注意：在 CSS Position 屬性當中， Sticky 屬性算是最新的一個。主流瀏覽器從 2021 年之後才開始完整支援，IE 完全不支援。

四、定位偏移屬性

用來配合 position 使用（static 除外）：


top: 值;      /* 距離頂部 */
right: 值;    /* 距離右側 */
bottom: 值;   /* 距離底部 */
left: 值;     /* 距離左側 */

五、z-index（層疊順序）

可以想像成 Photoshop 的圖層概念，用來控制被定位的 HTML 元素的堆疊順序（只對已設定定位的元素有效）。


.layer-1 {
  position: absolute;
  z-index: 1;
  background-color: red;
}

.layer-2 {
  position: absolute;
  z-index: 2;
  background-color: blue;
  /* 會在 layer-1 上方 */
}

.layer-3 {
  position: absolute;
  z-index: 3;
  background-color: green;
  /* 會在 layer-2 上方 */
}

特性：

  • 數值越大，越在上層
  • 預設值為 auto（等於 0）
  • 可以使用負值

六、常見錯誤與注意事項

錯誤 1: inline 元素設定寬高

錯誤示範：


/* ❌ inline 元素無法設定寬高 */
span {
  display: inline;
  width: 200px;   /* 無效 */
  height: 100px;  /* 無效 */
}

正確做法：


/* ✅ 改用 inline-block */
span {
  display: inline-block;
  width: 200px;
  height: 100px;
}

錯誤 2: absolute 定位沒有參考點

錯誤示範：


/* ❌ 父元素沒有設定 position */
.parent {
  /* 沒有 position 屬性 */
}

.child {
  position: absolute;
  top: 20px;
  /* 會相對於 body 定位，而非 parent */
}

正確做法：


/* ✅ 父元素設定 position: relative */
.parent {
  position: relative;  /* 作為參考點 */
}

.child {
  position: absolute;
  top: 20px;
  /* 相對於 parent 定位 */
}

錯誤 3: z-index 對非定位元素無效

錯誤示範：


/* ❌ static 元素無法使用 z-index */
.box {
  position: static;  /* 或未設定 position */
  z-index: 999;      /* 無效 */
}

正確做法：


/* ✅ 必須是定位元素 */
.box {
  position: relative;  /* 或 absolute, fixed, sticky */
  z-index: 999;
}

錯誤 4: fixed 元素導致內容被遮蓋

問題示範：


/* ❌ 固定導覽列遮住頁面內容 */
.header {
  position: fixed;
  top: 0;
  height: 60px;
}

/* 內容被遮住 */

正確做法：


/* ✅ 為內容留出空間 */
.header {
  position: fixed;
  top: 0;
  height: 60px;
}

body {
  padding-top: 60px;  /* 與 header 高度相同 */
}

錯誤 5: sticky 定位沒有設定 top / bottom / left / right

錯誤示範：


/* ❌ sticky 必須設定 top/bottom 等閾值 */
.sticky {
  position: sticky;
  /* 缺少 top, bottom, left, 或 right */
}

正確做法：


/* ✅ 設定觸發閾值 */
.sticky {
  position: sticky;
  top: 0;  /* 捲動到頂部時固定 */
}

七、重點整理

Display 的選擇：

  • 需要獨佔一行 → block
  • 需要在同一行且設定尺寸 → inline-block
  • 純文字或不需折行的小元素 → inline

Position 的選擇：

  • 微調位置 → relative
  • 相對於父元素定位 → absolute（父元素要設定 position）
  • 固定在視窗 → fixed
  • 捲動時有變化 → sticky

Absolute 定位的參考點：

  • 向上尋找第一個非 static 的父元素
  • 找不到就相對於 body
  • 通常在父元素設定 position: relative

z-index 的使用：

  • 只對定位元素有效
  • 同一層級才會比較
  • 會相對於視窗，不受捲動影響

Fixed 定位的影響：

  • 脫離文檔流，不佔空間
  • 記得為其他內容留出空間
  • 建議全域設定為 border-box

Sticky 的瀏覽器支援：

  • 現代瀏覽器普遍支援
  • IE 不支援
  • 需要設定（top / bottom / left / right）
