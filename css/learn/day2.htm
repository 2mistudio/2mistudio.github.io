Day 2: CSS 色彩空間介紹

CSS Color Space

一、學習目標

完成本日課程後，你將能夠：

  • 理解網頁設計中的色彩基礎概念
  • 掌握 CSS 的各種顏色表示法（顏色名稱、HEX、RGB、RGBA、HSL）
  • 了解色彩的透明度與應用場景
  • 設定文字與背景顏色
  • 建立基本的配色方案
  • 運用色彩對比提升可讀性

二、色彩基礎概念

色彩是網頁設計中最具影響力的元素之一，能夠：

  • 傳達情感和氛圍
  • 建立視覺層級
  • 引導使用者注意力
  • 強化品牌識別

色彩的三個基本屬性：

  • 色相 (Hue)：顏色的種類（紅、藍、綠等）
  • 彩度 (Saturation)：又稱作飽和度，是顏色的純度或鮮豔程度
  • 明度 (Lightness/Brightness)：又稱作亮度，顏色的明暗程度

三、CSS 顏色表示法

1. 顏色名稱 (Color Names)

CSS 支援 140+ 個預定義的顏色名稱。

常用顏色：

  • 基本色：red, blue, green, yellow, orange, purple, pink
  • 黑白灰：black, white, gray, lightgray, darkgray
  • 特殊色：transparent（完全透明）

優點： 容易記憶、語意清楚、可以快速使用
缺點： 選擇有限、不夠精確、沒有設計感

2. HEX 十六進位 (Hexadecimal)

使用 # 開頭，後接 6 位十六進位數字。

格式： #RRGGBB

  • RR = 紅色 (00-FF)
  • GG = 綠色 (00-FF)
  • BB = 藍色 (00-FF)

簡寫規則：當六位數字兩兩相同時，可簡寫為三位：

  • #ff0000 → #f00（紅色）
  • #00ff00 → #0f0（綠色）
  • #0000ff → #00f（藍色）
  • #ffffff → #fff（白色）
  • #000000 → #000（黑色）

優點： 最常用、簡潔、標準化、設計工具普遍支援
缺點： 不直觀、難以理解色值含義、調整顏色麻煩

備註：

  • 電腦上的十六進位制（Hexadecimal）指的是由 0 到 F 共計十六位。平時數數是用「十進位」（0 到 9），但十六進位需要 16 個符號。因為數字只有 0 到 9，所以剩下的 6 個數值就用英文字母 A 到 F 來代替。
  • RR 有兩個空格，每一個空格可以有十六種變化，兩個空格（16 x 16）就可以衍生出 256 種變化。

3. RGB (Red, Green, Blue)

使用紅、綠、藍光線三原色的數值組合。

語法：


rgb(紅, 綠, 藍);

/*範例*/
rgb(255, 0, 0);

三原色混合原理：

  • rgb(255, 0, 0) = 紅色
  • rgb(0, 255, 0) = 綠色
  • rgb(0, 0, 255) = 藍色
  • rgb(255, 255, 255) = 白色（全開）
  • rgb(0, 0, 0) = 黑色（全關）

優點： 直觀易懂、容易調整單、與設計軟體的色彩選擇器對應
缺點： 語法較長、顏色細部調較不容易

4. RGBA (RGB + Alpha)

在 RGB 基礎上增加透明度通道（Alpha Channel）。


rgb(紅, 綠, 藍, 透明度)

/*範例*/
rgb(255, 0, 0, 0.5);

透明度範圍： 0-1 之間（也可以用百分比設定）

  • 0 = 完全透明
  • 0.5 = 50% 透明
  • 1 = 完全不透明

使用場景：

  • 半透明背景
  • 層疊效果
  • 遮罩效果
  • 漸層設計（從不透明漸層到透明）

重要觀念： RGBA 的透明度不會影響子元素，只影響背景本身。

5. HSL (Hue, Saturation, Lightness)

基於色相、飽和度、亮度的色彩模型，更符合人類對顏色的認知。

語法：


hsl(色相, 飽和度%, 亮度%);

/*範例*/
hsl(150, 50%, 90%);

優點： 調整顏色更直觀、容易創建色彩細微變化（同色系深淺）、適合建立配色系統方案
缺點： 語法相對較新，部分開發者可能不熟悉

6. HSLA (HSL + Alpha)

在 HSL 基礎上增加透明度通道。

語法：


hsl(色相, 飽和度%, 亮度%, 透明度);

/*範例*/
hsl(150, 50%, 90%, 50%);

透明度範圍同 RGBA（0-1 或 %）。

四、常見錯誤與注意事項

錯誤 1: HEX 格式錯誤

錯誤示範：


/* ❌ 缺少 # 符號 */
.bad {
  color: 333333;
}

/* ❌ 位數不對（HEX 只能是 3 位或 6 位） */
.bad {
  color: #3333;
}

/* ❌ 使用了非十六進位字元（G-Z 不是有效字元） */
.bad {
  color: #gghhii;
}

正確做法：


/* ✅ 正確格式 */
.good {
  color: #333333;
}

/* ✅ 簡寫格式 */
.good {
  color: #333;
}

錯誤 2: RGB/RGBA 數值超出範圍

錯誤示範：


/* ❌ RGB 數值超過 255 */
.bad {
  color: rgb(300, 400, 500);
}

/* ❌ Alpha 透明度超過 1 */
.bad {
  background-color: rgba(100, 100, 100, 1.5);
}

正確做法：


/* ✅ RGB 範圍 0-255 */
.good {
  color: rgb(255, 255, 255);
}

/* ✅ Alpha 範圍 0-1 */
.good {
  background-color: rgba(100, 100, 100, 0.5);
}

錯誤 3: HSL 格式錯誤

錯誤示範：


/* ❌ 飽和度和亮度缺少 % 符號 */
.wrong {
  color: hsl(210, 100, 50);
}

/* ❌ 色相超過 360 度（雖然技術上可以，但不規範） */
.wrong {
  color: hsl(400, 100%, 50%);
}

正確做法：


/* ✅ 飽和度和亮度必須加 % */
.correct {
  color: hsl(210, 100%, 50%);
}

/* ✅ 色相建議保持在 0-360 範圍 */
.correct {
  color: hsl(40, 100%, 50%);  /* 400 - 360 = 40 */
}

注意事項

1. 選擇適合的顏色表示法

  • 設計階段：HSL（容易調整）
  • 生產環境：HEX（簡短乾淨）
  • 需要透明度：RGBA 或 HSLA

2. 保持一致性

  • 整個專案使用相同的顏色表示法
  • 建立色彩系統並記錄

3. 測試不同場景

  • 白天模式 / 夜間模式
  • 不同螢幕亮度
  • 色盲模擬工具測試

五、重點整理

CSS 顏色表示法：

  • 顏色名稱 - 簡單但選擇有限（red, blue, green 等）
  • HEX - 最常用，格式 #RRGGBB 或簡寫 #RGB
  • RGB - 直觀，rgb(r, g, b)，範圍 0-255
  • RGBA - RGB + 透明度，rgba(r, g, b, a)，透明度 0-1
  • HSL - 色相/飽和度/亮度，hsl(h, s%, l%)
  • HSLA - HSL + 透明度，hsla(h, s%, l%, a)

格式要點：

  • HEX 必須以 # 開頭
  • RGB/RGBA 數值範圍 0-255
  • HSL/HSLA 的飽和度和亮度要加 %
  • 透明度範圍 0-1

常用屬性：

  • color - 設定文字顏色
  • background-color - 設定背景顏色

選擇建議：

  • 日常使用：HEX（簡潔、通用）
  • 需要透明：RGBA 或 HSLA
  • 調整色調：HSL（直觀）

延伸閱讀：

  • 現代 CSS 還支援 OKLCH、LCH、Lab 等進階色彩空間
  • 了解色彩對比度與無障礙設計（WCAG 標準）
  • 學習配色理論（互補色、類似色、三角配色等）
