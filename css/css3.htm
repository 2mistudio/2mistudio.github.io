關於 CSS 3

CSS 3 是 CSS 2 的後繼版本。最早從 1999 年就開始規劃。 由於在 CSS 2 中已經定義了各種詳細的規範，內容相當繁瑣。若要延續 CSS 2 的方式來開發 CSS 3 ，恐怕需要花費更多時間，無法因應快速發展的網路技術需求。

因此 CSS 3 是以 CSS 2.1 規範為核心，分拆成為逐一模組 ( Module ) 的方式開發，在 CSS 2 的基礎上進行擴展 ( 例如 Selectors Module, Box model Module, Color Module, Fonts Module ⋯ 等等 )，讓新功能可以更快速的進展跟發布。每個 Module 有各自的名稱跟層級編號，也會增加新功能，或是替換掉 CSS 2.1 規範中的部分內容。

CSS 3 的 Level 層級編號系統

從 CSS3 開始，各個模組的層級 ( Level ) 是獨立演進的：例如，「Selectors Module Level 4」很有可能會比「Line Module Level 3」提前定案。

  • Level 1: 如果是過去未曾出現的全新功能，在 CSS 3 中會以 Level 1 作為起始層級編號 ( 例如 Flexbox Module Level 1 或是 Grid Module Level 1 )。
  • Level 3: 如果這個功能在 CSS 2.1 就已經存在，那麼在 CSS 3 會以 Module Level 3 作為起始層級編號 ( 像是 Color Module Level 3 跟 Fonts Module Level 3 )。
  • Level 4 或以上: 如果 Level 3 版本已經通過 W3C REC 標準審核，那下一個版本或增加的新功能就會從 Level 4 開始作區隔 ( 例如 Selectors Module Level 4 )。

CSS 3 的 Snapshots 快照系統

由於技術上 CSS 3 這個名詞已經是個終點 ( 因為從它開始 CSS 被分拆成許多分支模組 )，所以如果統稱 CSS 3 無法明確的定義用的是哪個階段哪個時期的 CSS 技術。為了解決這個困擾，W3C 推出了Snapshots 快照系統。

快照系統中定義了在特定時間哪些模組是穩定且可以使用的 ( 例如 CSS Snapshot 2015，就定義了在 2015 年技術上成熟可用的 CSS 模組)

  • CSS 2.1 - 完善了整個 CSS 的核心結構。
  • CSS Snapshot 2010 - 首次嘗試將規劃完善的 CSS Module 集合起來。
  • CSS Snapshot 2015 - 加入了 CSS Flexbox 跟更新 Background / Border 等的新規則。
  • CSS Snapshot 2020 - 加入了 CSS Grid 跟 Custom Properties (CSS Variables)。
  • CSS Snapshot 2023 - 著重在更彈性的排版功能像是 Container Queries，Nesting 跟 Subgrid
  • CSS Snapshot 2025 - 包含了 Anchor Positioning 與 View Transitions。
