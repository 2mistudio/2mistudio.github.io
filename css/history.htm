CSS的發展歷程

形成時期 (1994 - 1995)

  • 1994年: 來自挪威的電腦科學家 Håkon Wium Lie (哈肯·維姆·萊) 在歐洲核子研究組織 (CERN) 與網際網路的發明者 Tim Berners-Lee 共事期間，於 10 月 10 日首次提出了 「階層式 HTML 樣式表」(Cascading HTML Style Sheets - CHSS) 的概念。
  • 1995年: Lie 與荷蘭挪威電腦科學家 Bert Bos (伯特·波斯)共同發表了該概念的修訂版。Lie 將修訂後的 CSS 整合至自己開發的 Arena 瀏覽器 中作為測試平台，隨後 W3C 將 CSS 開發列為核心交付成果之一。

標準化與早期實作 (1996 - 1999)

  • 1996年: CSS 第一版 (CSS 1) 於 12 月 17 日正式發佈為 W3C 推薦標準。同年，微軟的 Internet Explorer 3 發表，並對該標準提供了有限的早期支援。
  • 1997年: 為了應對日益增長的複雜性，W3C 成立了專門的 CSS 工作小組 ( CSSWG - CSS Working Group )。
  • 1998年: CSS 第二版 (CSS 2) 於 5 月 12 日發佈。同年註冊了 text/css MIME 類型。
  • 1999年: 開始規劃 CSS 第三版 (CSS 3)，將 CSS 的細部功能分開成各自獨立的「模組 Module」形式開發，而非如同 CSS 2 一般的大型版本規格書註。

在不同瀏覽器之間的互通與掙扎 (2000 - 2009)

  • 2000年: 微軟推出的 Macintosh 版 Internet Explorer 5.0 成為首個實現近乎完整（99%）CSS1 支援的瀏覽器，但 Windows 版 IE 的支援度卻大相徑庭。
  • 2001年: 第一個 「CSS 3 模組」草案 The Box Model Working Draft 發佈。
  • 2003年: 蘋果推出第一個自行開發的瀏覽器 Safari，並超前支援大部分的 CSS 2.1 標準。
  • 2004年: CSS 2.1 達到「CR 候選推薦」(W3C Candidate Recommendation)狀態，主要修正原始 CSS2 中的錯誤。同年十一月 Mozilla 公司開發的自由開源網頁瀏覽器 Firefox 正式推出，並完整支援 CSS 1 標準。
  • 2005年: 為了加速推動瀏覽器開發商遵循 CSS 標準， Lie 針對微軟的 IE 發起了 Acid2 測試。同年四月蘋果 Safari 2.0 推出，成為首個通過該測試的瀏覽器。
  • 2006年: Opera 9 瀏覽器通過 Acid2 測試。 Mozilla 終止開發 Mozilla Suite (包含其中的瀏覽器)，Firefox 成為旗下的唯一瀏覽器。
  • 2007年: 由於 CSS 3 不再以版本為更新命名，為了方便瀏覽器開發商與網路開發者更容易的分辨 CSS 的發展狀態，W3C 轉向發佈快照 ( CSS Snapshots )，用來總結哪些 CSS 模組當前是穩定可用的。
  • 2008年: Google 推出瀏覽器 Google Chrome，在第一款測試版中 Chrome 就已經順利通過 Acid1 及 Acid2 測試，並且在 Acid3 測試中取得高分。同年 Firefox 3.0 也正式通過 Acid2 測試。
  • 2009年: 微軟 Internet Explorer 8 通過 Acid2 測試，並支援大部分的 CSS 2.1 標準以及少部分的 CSS3，市場上的主要瀏覽器終於實現能夠互通的 CSS 的願景，結束了長期以來不同瀏覽器顯示結果不一致的問題。

模組化時代與現代網路 (2010年至今)

  • 2011年: 可以說是 CSS 3 的元年。除了 CSS 2.1 終於正式成為「REC 完整推薦標準」(W3C Recommendation) 之外。CSS3 Color Module Level 3 也成為第一個通過「REC 完整推薦標準」的新 CSS 3 模組。
    從此時開始 CSS 3 各種功能模組開發飛快的進展，各家瀏覽器 ( 主要是 Chrome，Firefox 跟 Safari ) 也開始快速的追趕 CSS 3 的新標準。
  • 2012年: CSS 3 的 Media Queries Level 3 成為「REC 完整推薦標準」，象徵了響應式網頁時代正式展開。
  • 2014年: 網絡社群慶祝 Lie 提出首個 CSS 提案滿 20 週年。
  • 2020年: 隨著 Snapshots 版本越來越多，也越來越容易混淆，開發人員一直在思考是否要重新給 CSS 版本一個清楚的定義。W3C CSS Working Group 於是成立了一個 CSS4 社群小組，開始討論是否有需要重新界定 CSS 的新版本，及如何為開發者建立一個具版本參考價值的文檔。
