關於 Safari 的故事

和比爾蓋茲之間的交易

1997年，Steve Jobs 正式回歸蘋果，微軟的 Bill Gates 在 舊金山 Macworld 博覽會，對當時正面臨破產邊緣的蘋果投資了一億五千萬美元。 微軟並承諾在未來五年內將持續為 Mac 平台開發並發布與 Windows 版本對等的 Microsoft Office 軟體 ( 微軟這麼做一方面也是為了擺脫美國司法部的反壟斷調查 ）。

作為回饋，Steve Jobs 同意將 Internet Explorer (IE) 作為 Mac 的預設瀏覽器，藉此幫助微軟在當時與 Netscape 的瀏覽器戰爭中取得優勢；蘋果並撤銷了對微軟長期以來的專利侵權訴訟，雙方同意一項為期五年的廣泛專利交叉授權協議。

蘋果的 Mac 作業系統上一直沒有自己開發的瀏覽器 ( 早期的預設瀏覽器是 Netscape Navigator )。也因此 Internet Explorer 順理成章的成為 1997 年之後 - 2003 年之間 Mac 作業系統的預設瀏覽器。

原生的殺手級應用程式

隨著時間的推移，微軟在瀏覽器市場上的市佔率已接近頂點 (全球市占率超過 90%) ，對瀏覽器市場也越來越不用心。協議到期時間日漸逼近，蘋果面臨了需要有自己瀏覽器替代方案的壓力。

當時第一次瀏覽器大戰剛結束，市場上除了微軟的 IE 沒有太多穩定的瀏覽器可以選擇，蘋果剛砍掉重練的新一代作業系統 MacOS X 也才剛上市沒多久，還沒站穩腳步，急需要原生的殺手級應用程式。與其重新建構一款自己的瀏覽器，不如找現成合適的開源瀏覽器核心當作起點。

靈魂人物的加入：Dave Hyatt

2022 年，蘋果延攬了 Mozilla 負責 Phoenix 瀏覽器 ( Firefox 的前身 ) ，以及開發 Mac 版 Phoenix ( 叫做 Chimera/Camino 瀏覽器 ) 的工程師 David Hyatt 加入瀏覽器專案。

在當時 Phoenix 瀏覽器所使用的開源 Gecko 繪圖排版引擎頗受好評，搭配 MacOS X 原生工具所開發的 Camino 更是號稱 Mac 史上最好用的瀏覽器。但蘋果最後並沒有選擇 Mozilla 的 Gecko 瀏覽器引擎，取而代之參考了 Unix 電腦上知名的開源介面軟體 KDE ，裡頭的繪圖排版引擎 KHTML 以及 JavaScript 引擎 KJS ， 開發出自己的瀏覽器核心 WebKit。這個核心技術至今仍是所有 Apple 裝置上瀏覽器（包括 iOS、iPadOS 和 macOS 上的 Safari）的基礎。

Safari 瀏覽器的誕生

2003 年 1 月 7 日，Steve Jobs 在 Macworld 大會上正式發布了 Safari 的公開測試版，宣稱它是 Mac 平台上「 迄今為止最快的瀏覽器」。同年稍晚，隨著 MacOS X 10.3 Panther 的發布，Safari 取代 Internet Explorer 成為 Mac 的預設瀏覽器。

Chrome 的核心引擎

因為 WebKit 的核心源自於開源，蘋果某方面也延續這樣的精神，將 WebKit 瀏覽器核心計劃部分開放源始碼授權。因此當 Google 開始打算建構自己的瀏覽器時，一開始就選中了 WebKit 當作其核心引擎，雖然後來因為開發方向不同，Google 再以 WebKit 為基礎分支衍伸出自己的瀏覽器核心引擎 Blink。

後記

如同微軟一般，大軟體公司的業務眾多，瀏覽器畢竟只是眾多軟體計劃中的一小部分。隨著 MacOS 與 iOS 日漸成熟，Safari 市占率跟著與日俱增。蘋果也在 Safari 的開發上逐漸放慢了腳步。反倒是當初從 WebKit 分支出去的 Chrome 大放異彩，穩站市佔率第一名的寶座。

蘋果將習慣 Safari 與 macOS/iOS 系統綑綁，一年僅一次大更新，加上對新的 Web API 支援不佳，一度被封為「瀏覽器界的新 Internet Explorer」。

2020 年開始，蘋果面對法規壓力與市場競爭，開始顯著加快 Safari 的更新步調。除了大幅縮短功能更新週期，並開始與 Google、Microsoft、Mozilla 共同推動 Interop 計劃。加強網路標準的相容性與強化隱私跟安全性，重新將 Safari 瀏覽器塑造成跟的上時代的上網工具。
