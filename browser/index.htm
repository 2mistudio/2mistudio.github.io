關於瀏覽器

對網頁開發者來說，瀏覽器不僅僅只是一個「上網用的工具」，它更像是網頁作品的畫布、翻譯者甚至最後的審查官。網頁開發者一定要在瀏覽器開啟並測試過網頁內容，才能交付成品；精確一點來說，必須要在客戶或大多數使用者所使用的瀏覽環境中測試沒問題才能讓網頁上線。

舉例來說，早期在 WYSIWYG (所見即所得) 工具盛行的時候，設計師往往習慣在網頁編輯工具 ( 例如 Dreamweaver ) 裡確認結果，但設計完成之後，網站在瀏覽器中打開，卻發現了許多設計的時候看不到的問題。

再舉一個更常遭遇到的例子。網頁設計好了之後，設計師在自己的電腦上用瀏覽器打開測試沒問題，但網頁傳到了客戶手上，用客戶的電腦打開卻出現問題。經過一番來回討論溝通之後，發現客戶用的瀏覽器跟設計師用的不同。

從以上的兩個例子，我們可以發現瀏覽器對「網頁設計師」及「網頁開發者」的重要性。身為一個設計／開發人員，不僅需要清楚網頁設計的基本技術，甚至對網頁運行的環境 ( 瀏覽器 )，也必須有相當程度的認知與了解。

瀏覽器的核心引擎

引擎是瀏覽器最重要的核心，為了完美執行 HTML、CSS 和 JavaScript 程式代碼，甚至圖片、聲音、影片⋯等等多媒體，瀏覽器必須具備「排版引擎」以及「邏輯引擎」。

「排版引擎」負責將程式碼轉化為肉眼可見的「視覺畫面」。它會將 HTML、CSS 程式碼繪製成一般人看得懂的內容。

「邏輯引擎」負責「執行邏輯」。它會將 JavaScript 編譯成電腦可以直接執行的機器碼。

因為不同的瀏覽器使用的引擎不同，這導致了所謂的「相容性問題」。同樣的 CSS 代碼，在 Chrome 看起來很完美，在 Safari 上就有可能會跑版。

瀏覽器的版本與市占率

除了引擎不一樣會有「相容性問題」之外，瀏覽器的版本對網頁內容的顯示也有很大的影響。瀏覽器推出後，會陸續有「安全性更新」以及「功能性更新」。「功能性的更新」除了瀏覽器本身功能增加之外，更多是為了支援新的網頁技術，像是 HTML、CSS 和 JavaScript 的新功能等等。

因此設計網頁時必須要先確認你用的技術有哪些瀏覽器支援，哪些不支援；換個角度來說，網頁開發人員必須隨時關注瀏覽器的 「市占率狀況」，還有各個 「版本與支援的功能」。

那麼，要如何獲取瀏覽器的發展，跟決定那些功能可以使用呢？我們一般會參考下面三個方向：

  • 專案的用戶資料 ( Google Analytics )
    一般商業客戶通常會在網站上安裝 Google Analytics ( 又稱 GA) ，透過 GA 後台的技術分析資料，可以精準的了解網站受眾透過什麼裝置上網，使用哪種瀏覽器與上網裝置。是最精準的重要參考資料。
  • 全球與地區市佔率 ( StatCounter )
    透過 StatCounter 網站，可以免費查詢全球瀏覽器跟上網裝置的分布現況。還可以細分到每個地區與國家 (因為分布狀況有可能不同)。
  • 技術支援查詢 ( Can I Use )
    Can I Use 是網頁開發者必備的「技術百科」網站，透過數據顯示目前各項 HTML/CSS/JS 等網路技術，在不同瀏覽器的支援度。使用者除了參考紅、綠色塊來理解各家瀏覽器的相容性之外。也可透過 Basekine 標記 來快速判斷這些新技術使已經否穩定可用。
