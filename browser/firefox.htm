Firefox 火狐狸誕生的故事

早期的瀏覽器非常不穩定，閃退、當機、或是網頁載入不完全，是家常便飯。後來 Internet Explorer 幾乎完全壟斷整個瀏覽器市場，其對於網頁技術的支援落差與對新標準的無視，更是造成網頁開發人員與使用者的困擾。

Firefox 在 2003 年的橫空出世。主打著穩定、簡單、輕便，加上「分頁瀏覽」、「內建搜尋列」、「攔截彈出式視窗」⋯種種在當時令人耳目一新的功能，在那個 IE 市占率大到可以為所欲為的時代，Firefox 相對著重隱私，比較不商業化，同時對網頁技術的支援也較完整跟跟嚴謹，無疑是荒漠中的甘泉。因此快速獲得不少 網頁開發人員跟著重隱私的使用者所喜好。

行動代號：鳳凰計劃

在 1998 年 Netscape 將程式碼開源後， Mozilla 組織成立。當時領導層的目標希望延續 Netscape 的願景，打造一個「全功能通訊套件」才能吸引企業用戶，跟 Microsoft Office 與 Internet Explorer 分庭抗禮。因此， Mozilla 瀏覽器被定位成 Mozilla Suite。它不只是一個瀏覽器，還包含了一個 Email 軟體、一個 IRC 聊天室和一個 HTML 編輯器。

當時 Mozilla 的開發工程師 David Hyatt 和實習生 Blake Ross 並不認同。他們認為瀏覽器應該要「輕快且專注」，而不是一個臃腫的萬能百寶箱。於是，Hyatt 和 Ross 在沒有得到公司正式批准的情況下，開始了一個代號 "Phoenix"（鳳凰） 的實驗性專案。

專案的目標，是利用 Mozilla 的 Gecko 排版引擎，加上 XUL 介面架構，重新組合成一個輕巧、靈活且讓第三方開發者也能參與功能擴充的新瀏覽器。

因為 Phoenix 不是公司正式的專案，所以 Mozilla 一開始採取觀望甚至懷疑的態度在看待這件事情。直到2003 年，Mozilla 背後的金主 AOL ( 美國線上 Ameria Online ) 因為網路泡沫化的影響，決定撤資並遣散了大部分開發人員。導致 Mozilla 基金會必須自負盈虧。因此，Mozilla 領導層做出了一個大膽的決定——宣佈放棄 Mozilla Suite 作為核心產品，將開發重心全面轉向瀏覽器 Firefox (當時名為 Firebird ) 和收信軟體 Thunderbird 。

2004 年 Firefox 1.0 發佈後的成功，證明了這個「鳳凰計劃」的方向是正確的。不到一年，Firefox 瀏覽器就拿下了近 10% 的市場份額，這是 Mozilla 組織成立以來從未有過的榮景。Firefox 當時帶來的技術創新，像是「分頁瀏覽」、「內建搜尋列」、「攔截彈出式視窗」、「使用者可自行安裝的擴充套件生態系」⋯等，都成為現代瀏覽器的標準配備。

瀏覽器的命名

母公司 Mozilla 基金會的命名靈感源自 Netscape 最早的瀏覽器 Mosaic 加上日本的知名神獸怪物 Godzilla，因此鳳凰計劃一開始也延續這個概念，以神話生物作為命名 備註。Firefox 一開始原本叫做 Phoenix (鳳凰)，代表它是從 Netscape Navigator 灰燼中誕生的瀏覽器。

2003 年，因為 Phoenix 的商標早被知名的 BIOS 技術公司鳳凰科技（ Phoenix Technologies ）註冊，為了避免商標侵權，開發小組將 Phoenix 改名為同樣意思的 Firebird ( 火鳥 )。

Mozilla 基金會曾向各界保證，該瀏覽器將永久沿用 Mozilla Firebird 之名以避免混淆。後來卻又發現與 Firebird 資料庫專案撞名。 在面臨更多壓力後，Mozilla Firebird 最終於 2004 年 2 月 9 日正式更名為 Mozilla Firefox。Firefox 這個名字源自小熊貓（Red Panda）的暱稱，而小熊貓隨後也成為了該新專案的吉祥物。

備註：Phoenix 的雙胞胎，另一款同樣用 Gecko 核心，但是用 Mac OS X 原生語言 Objective-C 跟 Cocoa 編譯的瀏覽器 Chimera 奇美拉( 後來改名叫 Camino )，也是採用神獸的命名。

Firefox 的生存危機

誰都沒料到，雖然市佔率不高，但一直以來但總是穩居老三老四地位的 Firefox，在 2024 年迎來了一場「生死存亡」的危機。這場危機並非來自於技術失敗，而是來自於它最主要的金主—— Google。

Mozilla 基金會不是一個營利機構，主要營運收入來源倚賴各方的贊助。諷刺的是，Firefox 頭號勁敵， Chrome 的母公司 Google， 卻是 Mozilla 最大的贊助商。Google 每年支付數億美元 ( 約佔 Mozilla 收入的 80% 到 85% ) ，讓自己成為 Firefox 的預設搜尋引擎。

美國司法部（DOJ）控告 Google 透過支付巨額費用（給 Apple、Mozilla 等）來鎖定預設搜尋位置，藉此維持壟斷地位。雖然反壟斷是為了促進公平競爭，但在現實層面，這可能會先斷了 Firefox 的糧草。

到了 2025 年底，案件有了初步的方向。法官裁定 Google 不能再簽署「排他性」協議，也就是說 Google 不能要求合作夥伴「只能」放 Google 搜尋。只要合約不是排他的、允許使用者輕易切換，Google 依然可以付錢給 Mozilla；反過來說，如果 Firefox 市佔率不高，競爭力不夠，對 Google 贊助費的議價能力也會下降。

為了應對這種隨時可能斷糧的危機，Mozilla 在 2026 年開始推廣訂閱制、開發以隱私為核心的 AI 服務，甚至嘗試在瀏覽器內建更具隱私保護性的推薦廣告。這些多樣化的轉型是否能加速收入，幫助 Firefox 順利度過難關，再次浴火重生，就讓我們拭目以待了。
