免費網站空間申請說明

目前一般常見的「網站空間」可簡單分為兩種，一種是可申請網路空間上傳網站檔案或是自行部署相關平台機制，另一種是提供了現成的網站架構服務讓使用者可自行客製化的平台。

前者如付費的虛擬主機、雲端空間服務例如 Google Cloud 、 Amazon Web Service 或者像是 GitHub、Netlify、Vercel 等網站；後者如 Wix、Squarespace、Wordpress ⋯等等。

其中像是 GitHub、Netlify、Vercel 等網站，雖為付費的平台，但是針對小用量的基本使用者，皆提供了免費的網路服務，像是檔案原始碼上傳空間、免費網址、版本管控⋯等功能。

GitHub

GitHub 算是一個最普遍的線上軟體原始碼代管服務平台。它同時提供了 「付費帳戶」和 「免費帳戶」服務。幾乎所有的程式開發者都會申請一個 GitHub 帳號。它的最主要功能在存放網站或軟體的原始碼，以利於版本控管、備份儲存、檔案分享與共同協作。同時，它的 GitHub Pages 功能也提供了簡易的網頁代管服務，可存放「靜態的網頁」，同時提供一個簡單以 github.io 為結尾的網址。

GitHub Pages 用法

GitHub Pages 的設定相當簡單，只要透過以下步驟即可完成：

  • 在 GitHub 上面建立一個 Public 開放的 repository 專案
  • 再將專案名稱命名為 username.github.io
  • 建立或是上傳網頁內容（可以是單一檔案或是選取資料夾內多個檔案一起上傳）
  • 請注意首頁必須要以 index 為命名。例：index.htm
  • 隔一會之後直接打 username.github.io 網址即可連到該網站

※ 備註：username 即為您的 GitHub 帳號名稱。

Netlify & Vercel 使用注意事項

GitHub Pages 所提供的網站功能相當陽春，當檔案上傳到指定的位置之後就網站直接上線了。網址的部分也沒有給太多的選擇。如果使用者有進一步網站管理功能需求，例如：上線前預覽、自訂網址或轉址、Web 框架、其他進階 API 技術整合服務的話，Netlify 或是 Vercel 會是比較理想的網頁服務。

由於上述網站都可與 GitHub 功能做連結，因此如果已經註冊了 GitHub 帳號，就可以直接透過 GitHub 來「申請」與「登入」這兩個服務。所以還是建議申請 GitHub 帳號為優先。

請注意：這些網站服務的「免費服務方案」都有一定的流量限制。如果網站檔案較大（圖檔較多或是有使用影片或聲音檔案）擔心會將免費的流量快速用盡，會建議將檔案放置在 GitHub ，再用轉址的方式串連至 Netlify 或是 Vercel 即可。
