Day 3: 運算子與表達式

Knowing Operator and Expression

一、學習目標

完成本日學習後，你應該能夠：

  1. 理解運算子的種類與用途
  2. 正確使用算術運算子進行數學計算
  3. 使用比較運算子比較數值與資料
  4. 使用邏輯運算子組合條件判斷
  5. 區分 == 與 === 的差異並正確使用

二、核心概念解釋

什麼是運算子？

運算子（Operator）是程式語言中用來執行特定運算的符號。就像數學中的加減乘除符號，JavaScript 的運算子讓我們能夠對資料進行各種操作，包括計算數值、比較大小、組合條件等。

運算子作用的對象稱為「運算元」（Operand）。例如在 5 + 3 中，+ 是運算子，5 和 3 是運算元，而整個式子稱為「表達式」（Expression），最終會產生一個結果值。

算術運算子

算術運算子用於執行數學計算，是程式開發中最基礎的運算工具。JavaScript 提供了加（+）、減（-）、乘（*）、除（/）、取餘數（%）、以及次方（**）等常用算術運算子。其中取餘數運算子（%）在判斷奇偶數、週期性邏輯等情境中特別實用。

比較運算子

比較運算子用於比較兩個值的大小或是否相等，運算結果必定為布林值（true 或 false）。在條件判斷與迴圈控制中，比較運算子扮演相當關鍵的角色。

JavaScript 中有一個特別重要的概念：== 與 === 的差異。雙等號（==）在比較時會自動進行型別轉換，可能導致意外結果；三等號（===）則會同時比較值與型別，是更嚴謹的寫法，現代 JavaScript 開發中強烈建議優先使用 ===。

邏輯運算子

邏輯運算子用於組合多個條件，常見的有 AND（&&）、OR（||）、NOT（!）三種。當需要判斷「同時滿足多個條件」或「至少滿足一個條件」時，邏輯運算子就能派上用場。例如判斷一個使用者是否已登入且擁有管理員權限，就需要用 && 將兩個條件結合。

三、語法與使用方式

範例一：算術運算子


let a = 10;
let b = 3;

console.log(a + b);   // 13（加法）
console.log(a - b);   // 7（減法）
console.log(a * b);   // 30（乘法）
console.log(a / b);   // 3.3333...（除法）
console.log(a % b);   // 1（取餘數）
console.log(a ** b);  // 1000（次方：10 的 3 次方）

說明： 取餘數（%）會回傳除法後的餘數。例如 10 % 3 = 1，因為 10 ÷ 3 商 3 餘 1。次方（**）是 ES7 新增的語法。

範例二：遞增與遞減運算子


let count = 5;

count++;             // 等同於 count = count + 1
console.log(count);  // 6

count--;             // 等同於 count = count - 1
console.log(count);  // 5

說明： ++ (遞增)和 -- (遞減)是常用的簡寫，特別在迴圈計數時非常實用。


let userName = "小明";

範例三：比較運算子


let x = 10;
let y = 20;

console.log(x > y);    // false（大於）
console.log(x < y);    // true（小於）
console.log(x >= 10);  // true（大於等於）
console.log(x <= 9);   // false（小於等於）
console.log(x == "10"); // true（只比較值，不比較型別）
console.log(x === "10"); // false（同時比較值與型別）
console.log(x !== y);  // true（不等於）

說明： == 會將字串 "10" 自動轉換成數字 10 再比較，所以結果為 true。而 === 發現左邊是 number、右邊是 string，型別不同，結果為 false。

範例四：邏輯運算子


let isLoggedIn = true;
let isAdmin = false;

console.log(isLoggedIn && isAdmin);  // false（AND：兩者皆須為 true）
console.log(isLoggedIn || isAdmin);  // true（OR：至少一個為 true）
console.log(!isLoggedIn);            // false（NOT：反轉布林值）

說明： && 需要左右兩側都是 true 才回傳 true；|| 只要其中一側為 true 就回傳 true；! 則是將 true 變 false、false 變 true。

範例五：複合賦值運算子


let score = 100;

score += 20;   // 等同於 score = score + 20，結果 120
score -= 10;   // 等同於 score = score - 10，結果 110
score *= 2;    // 等同於 score = score * 2，結果 220
score /= 4;    // 等同於 score = score / 4，結果 55

console.log(score);  // 55

說明： 複合賦值運算子是簡化寫法，讓程式碼更簡潔易讀，在實務開發中非常普遍。

四、常見錯誤與注意事項

錯誤一：混用 == 與 ===

錯誤範例：


let age = "18";

if (age == 18) {
  console.log("成年");  // 會執行，但這是危險的判斷方式
}

問題： == 會自動轉換型別，容易造成邏輯漏洞。如果 age 來自使用者輸入，可能導致非預期的判斷結果。

正確寫法：


let age = "18"; //這邊的 "18" 是 String 字串，不是 Number 數字

if (age === 18) {
  console.log("成年");  // 不會執行，因為型別不同
}

最佳實踐： 永遠使用 === 和 !== 進行比較，避免隱性型別轉換帶來的意外問題。

錯誤二：除以零

錯誤範例：


let result = 10 / 0;
console.log(result);  // Infinity（無限大）

let result2 = 0 / 0;
console.log(result2);  // NaN（Not a Number）

說明： JavaScript 不會直接報錯，而是回傳特殊值。Infinity 和 NaN 都可能導致後續計算出現錯誤，需要事先做除數的驗證。

正確做法：


let divisor = 0;

if (divisor !== 0) {
  console.log(10 / divisor);
} else {
  console.log("除數不能為零");
}

錯誤三：字串與數字的加法混淆

錯誤範例：


let input = "5";  // 來自使用者輸入的字串
let bonus = 10;

console.log(input + bonus);   // "510"（字串連接，非數學運算）

正確寫法：


let input = "5";
let bonus = 10;

console.log(Number(input) + bonus);  // 15（先轉換型別再運算）
// 或
console.log(parseInt(input) + bonus);  // 15

錯誤四：運算子優先順序

容易混淆的範例：


console.log(2 + 3 * 4);   // 14，不是 20
console.log((2 + 3) * 4); // 20

說明： JavaScript 遵循數學的運算優先順序，乘除優先於加減。不確定優先順序時，使用括號明確指定計算順序是最安全的做法。

五、重點整理

  • 算術運算子：加（+）、減（-）、乘（*）、除（/）、取餘數（%）、次方（**），複合賦值（+=、-= 等）可簡化程式碼。
  • 永遠使用 ===：比較時使用三等號，避免 == 的隱性型別轉換帶來的邏輯錯誤。
  • 邏輯運算子：&&（AND）需兩側皆為 true；||（OR）至少一側為 true 即可；!（NOT）反轉布林值。
  • 注意特殊值：除以零會產生 Infinity 或 NaN，使用前需驗證除數。
  • 型別轉換陷阱：字串與數字相加會變成字串連接，需使用 Number() 或 parseInt() 明確轉換型別。
