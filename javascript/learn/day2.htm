Day 2: 變數與資料型別

Knowing Variable and Data Types

一、學習目標

完成本日學習後，你應該能夠：

  1. 理解變數的概念與用途
  2. 正確使用 let、const、var 宣告變數
  3. 區分 let、const、var 三者的差異
  4. 認識 JavaScript 的基本資料型別
  5. 使用 typeof 運算子檢查資料型別

二、核心概念解釋

什麼是變數？

變數是程式中用來儲存資料的容器。就像現實生活中的箱子，我們可以在箱子上貼上標籤（變數名稱），並在箱子裡放入物品（資料）。當需要使用這些資料時，只要透過箱子上的標籤就能找到正確的資料。

在 JavaScript 中，變數讓我們能夠暫時儲存並操作資料，例如使用者輸入的姓名、計算結果、網頁元素等。沒有變數，程式將無法記憶任何資訊，也就無法進行有意義的運算。

變數宣告的三種方式

JavaScript 提供三種宣告變數的關鍵字：let、const 和 var。

let - (Let 就是讓的意思）
用於宣告可以重新賦值的變數，具有區塊作用域（block scope），是現代 JavaScript 最常用的變數宣告方式。

const - (Constant 就是經常，恆常的意思）
用於宣告常數，一旦賦值後就不能再改變，同樣具有區塊作用域。適合用於宣告不會改變的值，如設定檔參數、固定配置等。

var - (Variable 就是變數的意思）
是較舊的變數宣告方式，具有函式作用域（function scope），在現代開發中較少使用，但理解其存在仍有必要，因為許多舊專案仍使用 var。

資料型別

JavaScript 是動態型別語言，變數的型別會根據所賦予的值自動決定。JavaScript 的基本資料型別包括：

Number（數字）：包含整數與浮點數，例如 42、3.14、-10。

String（字串）：文字資料，必須用引號包圍，例如 "Hello"、'JavaScript'。

Boolean（布林值）：邏輯值，只有 true（真）和 false（假）兩種。

Undefined：變數已宣告但未賦值時的預設值。

Null：表示「空值」或「無」的特殊值。

三、語法與使用方式

範例一：使用 let 宣告變數


let userName = "小明";
console.log(userName);  // 小明

userName = "小華";  // 重新賦值
console.log(userName);  // 小華

說明： let 宣告的變數可以重新賦值。第一次賦值後，後續可以改變變數的內容。

範例二：使用 const 宣告常數


const pi = 3.14159;
console.log(pi);  // 3.14159

pi = 3.14;  // 錯誤！const 不能重新賦值

**執行結果：**
Uncaught TypeError: Assignment to constant variable.

說明： const 宣告的常數一旦賦值就不能改變，嘗試重新賦值會產生錯誤。

範例三：資料型別 - Number


let age = 25;
let price = 99.9;
let temperature = -5;

console.log(typeof age);         // number
console.log(typeof price);       // number
console.log(typeof temperature); // number

說明： JavaScript 不區分整數與浮點數（近似小數點的意思），統一使用 number 型別。使用 typeof 運算子可以檢查變數的型別。

範例四：資料型別 - String


let firstName = "張";
let lastName = '三';
let greeting = "Hello, World!";

console.log(typeof firstName);  // string
console.log(firstName + lastName);  // 張三

說明： 字串可以使用單引號或雙引號。使用 + 運算子可以連接字串。

範例五：資料型別 - Boolean


let isStudent = true;
let hasLicense = false;

console.log(typeof isStudent);  // boolean
console.log(isStudent);         // true

說明： Boolean 又稱布林值，只有 true 和 false（開跟關），常用於條件判斷與邏輯運算。

範例六：Undefined 與 Null


let x;
console.log(x);         // undefined
console.log(typeof x);  // undefined

let y = null;
console.log(y);         // null
console.log(typeof y);  // object（這是 JavaScript 的歷史遺留問題）

說明： undefined （未定義）表示變數未賦值，null (沒有)表示刻意設定的空值。

四、常見錯誤與注意事項

錯誤一：使用未宣告的變數

錯誤範例：


console.log(myName);  // 變數未宣告就使用

**錯誤訊息：**
Uncaught ReferenceError: myName is not defined

正確寫法：


let myName = "小明";
console.log(myName);

錯誤二：嘗試改變 const 的值

錯誤範例：


const score = 100;
score = 95;  // 錯誤

正確做法：如果值需要改變，應使用 let 而非 const。


let score = 100;
score = 95;  // 正確

錯誤三：變數命名不規範

錯誤範例：


let 1name = "小明";      // 不能以數字開頭
let my-name = "小華";    // 不能使用 - 連字號
let my name = "小李";    // 不能有空格

正確寫法：


let name1 = "小明";
let myName = "小華";     // 駝峰式（camelCase）命名（推薦）
let my_name = "小李";    // 蛇形（snake_case）命名（較少用）

命名規則：

  • 只能包含字母、數字、底線、錢字號
  • 不能以數字開頭
  • 區分大小寫
  • 不能使用保留字（如 let、const、if 等）

錯誤四：型別混淆

容易混淆的範例：


let num1 = "10";     // 字串
let num2 = 10;       // 數字

console.log(num1 + num2);  // "1010" 字串連接，而非 20

說明： 當字串與數字相加時，JavaScript 會將數字轉換為字串進行連接，而非數學運算。

正確做法：


let num1 = 10;
let num2 = 10;
console.log(num1 + num2);  // 20

錯誤五：忘記宣告關鍵字

不良範例：


userName = "小明";  // 忘記使用 let 或 const

問題： 雖然程式可以執行，但會在全域作用域建立變數，可能導致意外的錯誤。

正確做法：


let userName = "小明";

五、重點整理

  • 變數宣告：使用 let 宣告可變動的變數，使用 const 宣告不可變的常數，避免使用舊式的 var。
  • 基本型別：JavaScript 有五種基本資料型別：Number、String、Boolean、Undefined、Null。
  • typeof 運算子：使用 typeof 可以檢查變數的資料型別。
  • 命名規範：變數名稱必須有意義、遵循命名規則、使用駝峰式命名法（camelCase）。
  • 最佳實踐：優先使用 const，只有在需要重新賦值時才使用 let；避免型別混淆導致的運算錯誤。
