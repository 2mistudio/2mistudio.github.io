Day 9: 建立與刪除元素

Create and Remove Element

一、學習目標

完成本日學習後，你應該能夠：

  1. 使用 JavaScript 動態建立新的 HTML 元素
  2. 將新建立的元素插入到網頁中
  3. 從網頁中移除元素
  4. 理解父元素、子元素的關係
  5. 製作動態清單與互動式介面

二、核心概念解釋

為什麼要動態建立元素？

前兩天我們學會了「選取」和「修改」現有的元素，但在實際開發中，我們經常需要「憑空創造」新元素，或是「移除」不需要的元素。

常見情境：

  • 待辦清單：使用者新增一筆待辦事項，需要建立新的清單項目
  • 購物車：使用者加入商品，需要建立新的商品卡片
  • 相簿：動態載入圖片，需要建立多個圖片元素
  • 訊息通知：系統發送通知，需要建立通知元素
  • 刪除功能：使用者點擊「刪除」按鈕，需要移除對應的元素

想像在 Figma 中：

  • 建立元素 = 在畫布上新增一個圖層
  • 插入元素 = 將圖層放到特定的 Frame 裡
  • 刪除元素 = 刪除一個圖層

DOM 操作的四個步驟

建立並插入一個新元素通常需要四個步驟：

  • 建立元素：document.createElement()
  • 設定內容/屬性：設定文字、class、樣式等
  • 選取父元素：找到要插入的位置
  • 插入元素：使用 appendChild() 或其他方法插入

父子關係的概念

在 DOM 樹中，元素之間有「父子關係」：


<div id="container">  <!-- 父元素 -->
    <p>段落 1</p>      <!-- 子元素 -->
    <p>段落 2</p>      <!-- 子元素 -->
</div>

就像 Figma 的圖層結構：
Frame (container)
├── Text (段落 1)
└── Text (段落 2)

要新增元素到網頁中，我們需要：

  • 找到「父元素」（要放在哪裡）
  • 建立「子元素」（要建立什麼）
  • 把子元素「插入」到父元素中

插入元素的方法

  • appendChild() → 插入到父元素的「最後面」
  • prepend() → 插入到父元素的「最前面」
  • insertBefore() → 插入到特定元素的「前面」
  • remove() → 移除元素本身

三、語法與使用方式

範例一：建立基本元素


// 步驟 1: 建立一個新的 div 元素
let newDiv = document.createElement("div");

// 步驟 2: 設定內容
newDiv.textContent = "這是一個新建立的 div";

// 步驟 3: 設定樣式或 class
newDiv.style.padding = "10px";
newDiv.style.backgroundColor = "#e3f2fd";
newDiv.style.borderRadius = "8px";
newDiv.style.margin = "10px 0";

// 步驟 4: 選取父元素
let list = document.querySelector("#list");

// 步驟 5: 插入到父元素中
list.appendChild(newDiv);

console.log("新元素已建立並插入！");

執行後，網頁上會出現一個新的藍色方塊，內容為「這是一個新建立的 div」。

說明： createElement() 只是在記憶體中建立元素，必須用 appendChild() 插入到網頁中才會顯示。

範例二：建立完整的清單項目


// 建立外層容器
let item = document.createElement("div");
item.className = "item";  // 設定 class

// 建立文字部分
let text = document.createElement("span");
text.textContent = "新的待辦事項";

// 建立刪除按鈕
let deleteBtn = document.createElement("button");
deleteBtn.textContent = "刪除";
deleteBtn.className = "delete-btn";

// 組裝：將文字和按鈕放入 item
item.appendChild(text);
item.appendChild(deleteBtn);

// 插入到清單中
let list = document.querySelector("#list");
list.appendChild(item);

console.log("清單項目已建立！");

執行後，會出現一個包含文字和刪除按鈕的項目。

說明： 這展示了如何建立「巢狀結構」：先建立多個元素，再組裝起來，最後一起插入到網頁中。

範例三：使用迴圈批次建立元素


let tasks = ["設計首頁", "撰寫程式碼", "測試功能", "上線部署"];
let list = document.querySelector("#list");

for (let i = 0; i < tasks.length; i++) {
    // 建立項目容器
    let item = document.createElement("div");
    item.className = "item";
    
    // 建立文字
    let text = document.createElement("span");
    text.textContent = (i + 1) + ". " + tasks[i];
    
    // 建立刪除按鈕
    let deleteBtn = document.createElement("button");
    deleteBtn.textContent = "刪除";
    deleteBtn.className = "delete-btn";
    
    // 組裝並插入
    item.appendChild(text);
    item.appendChild(deleteBtn);
    list.appendChild(item);
}

console.log("已建立 " + tasks.length + " 個項目");

執行後，會一次建立 4 個待辦事項。

說明： 實務上，我們通常從資料陣列動態產生 UI 元素，這在處理 API 資料時特別有用。

範例四：移除元素（remove）


// 先建立一個元素
let item = document.createElement("div");
item.className = "item";
item.textContent = "我會在 3 秒後消失";

let list = document.querySelector("#list");
list.appendChild(item);

// 3 秒後移除
setTimeout(function() {
    item.remove();  // 移除自己
    console.log("元素已移除");
}, 3000);

執行後，元素會出現 3 秒後自動消失。

說明： remove() 方法會將元素從 DOM 中完全移除。

範例五：點擊刪除功能（完整範例）


let addBtn = document.querySelector("#addBtn");
let list = document.querySelector("#list");
let itemCount = 0;

addBtn.addEventListener("click", function() {
    itemCount++;
    
    // 建立新項目
    let item = document.createElement("div");
    item.className = "item";
    
    // 建立文字
    let text = document.createElement("span");
    text.textContent = "項目 " + itemCount;
    
    // 建立刪除按鈕
    let deleteBtn = document.createElement("button");
    deleteBtn.textContent = "刪除";
    deleteBtn.className = "delete-btn";
    
    // 點擊刪除按鈕時移除整個項目
    deleteBtn.addEventListener("click", function() {
        item.remove();
        console.log("項目已刪除");
    });
    
    // 組裝並插入
    item.appendChild(text);
    item.appendChild(deleteBtn);
    list.appendChild(item);
    
    console.log("已新增項目 " + itemCount);
});

執行後，每次點擊「新增項目」會建立新項目，點擊項目的「刪除」按鈕會移除該項目。

說明： 這是待辦清單、購物車等功能的核心邏輯：建立元素並同時綁定刪除事件。

範例六：insertBefore（插入到特定位置）


let list = document.querySelector("#list");

// 建立新元素
let newItem = document.createElement("div");
newItem.className = "item";
newItem.textContent = "我會插入到最前面";

// 取得第一個子元素
let firstChild = list.firstElementChild;

// 插入到第一個子元素之前
if (firstChild) {
    list.insertBefore(newItem, firstChild);
} else {
    // 如果列表是空的，直接 appendChild
    list.appendChild(newItem);
}

說明： insertBefore(新元素, 參考元素) 會將新元素插入到參考元素之前。

範例七：使用 innerHTML 快速建立（簡便但較不安全）


let list = document.querySelector("#list");
let tasks = ["設計", "開發", "測試"];

// 方法一：使用 createElement（推薦，但較繁瑣）
// 如範例三所示

// 方法二：使用 innerHTML（快速，但要注意安全）
let html = "";
for (let i = 0; i < tasks.length; i++) {
    html += "<div class='item'>";
    html += "  <span>" + tasks[i] + "</span>";
    html += "  <button class='delete-btn'>刪除</button>";
    html += "</div>";
}
list.innerHTML = html;

說明： 使用 innerHTML 比較簡潔，但有兩個缺點：

  • 如果內容來自使用者輸入，可能有安全問題（XSS）
  • 會移除原有的事件監聽器

建議： 靜態內容可以用 innerHTML，需要綁定事件的動態內容建議用 createElement。

四、常見錯誤與注意事項

錯誤一：忘記插入到 DOM

錯誤範例：


let newDiv = document.createElement("div");
newDiv.textContent = "新元素";
// 忘記 appendChild，元素不會顯示在網頁上

問題： 元素只存在於記憶體中，沒有插入到 DOM，所以不會顯示。

正確寫法：


let newDiv = document.createElement("div");
newDiv.textContent = "新元素";

let container = document.querySelector("#container");
container.appendChild(newDiv);  // 必須插入

錯誤二：appendChild 會移動元素

容易混淆的範例：


let element = document.querySelector("#myElement");
let container1 = document.querySelector("#container1");
let container2 = document.querySelector("#container2");

container1.appendChild(element);  // 元素在 container1
container2.appendChild(element);  // 元素移動到 container2（不是複製！）

說明： appendChild() 會「移動」元素，不是「複製」。如果元素已經在 DOM 中，會先從原位置移除，再插入到新位置。

如果想複製元素：


let element = document.querySelector("#myElement");
let clone = element.cloneNode(true);  // true 表示深度複製（包含子元素）

let container2 = document.querySelector("#container2");
container2.appendChild(clone);  // 插入複製品

錯誤三：事件監聽器綁定時機錯誤

錯誤範例：


let deleteBtn = document.querySelector(".delete-btn");
deleteBtn.addEventListener("click", function() {
    // 錯誤：這時還沒有 .delete-btn 元素
});

// 之後才建立元素
let btn = document.createElement("button");
btn.className = "delete-btn";

問題： 元素必須先存在，才能綁定事件。如果元素是動態建立的，必須在建立後立即綁定事件。

正確寫法：


// 建立元素
let btn = document.createElement("button");
btn.className = "delete-btn";
btn.textContent = "刪除";

// 立即綁定事件
btn.addEventListener("click", function() {
    console.log("按鈕被點擊");
});

// 然後插入到 DOM
container.appendChild(btn);

錯誤四：innerHTML 會移除事件監聽器

錯誤範例：


let container = document.querySelector("#container");

// 先綁定事件
let btn = document.querySelector("#myBtn");
btn.addEventListener("click", function() {
    console.log("點擊");
});

// 使用 innerHTML 會移除所有子元素和事件
container.innerHTML = "";

// 原本的事件監聽器已經失效！

說明： innerHTML 會完全替換內容，原有的事件監聽器會消失。

解決方法： 如果需要動態建立元素並綁定事件，使用 createElement() 而不是 innerHTML。

錯誤五：在迴圈中重複選取父元素

效能不佳的寫法：


for (let i = 0; i < 100; i++) {
    let item = document.createElement("div");
    item.textContent = "項目 " + i;
    
    // 每次迴圈都選取一次（效能差）
    let list = document.querySelector("#list");
    list.appendChild(item);
}

比較好的寫法：


// 只選取一次
let list = document.querySelector("#list");

for (let i = 0; i < 100; i++) {
    let item = document.createElement("div");
    item.textContent = "項目 " + i;
    list.appendChild(item);
}

說明： DOM 查詢（querySelector）相對慢，應該在迴圈外執行，避免重複查詢。

五、重點整理

  • 建立元素：document.createElement("標籤名稱") 建立元素，但還不會顯示在網頁上。
  • 插入元素：
    • appendChild() → 插入到最後
    • prepend() → 插入到最前
    • insertBefore() → 插入到特定位置之前
  • 移除元素：element.remove() 從 DOM 中移除元素。
  • 設定內容與屬性：建立元素後，記得設定 textContent、className、style 等屬性。
  • 事件綁定時機：動態建立的元素，必須在建立後立即綁定事件，再插入到 DOM。
  • 效能考量：在迴圈外選取父元素，避免重複查詢 DOM。
