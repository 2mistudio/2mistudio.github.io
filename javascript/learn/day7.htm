Day 7: DOM 介紹與選取元素

Document Object Model and Element Selection

一、學習目標

完成本日學習後，你應該能夠：

  1. 理解 DOM（文件物件模型）的概念
  2. 認識 HTML、CSS 與 JavaScript 的關係
  3. 使用 JavaScript 選取網頁上的元素
  4. 理解不同選取方法的差異與使用時機
  5. 完成第一個真正的網頁互動效果

二、核心概念解釋

什麼是 DOM？

DOM（Document Object Model 文件物件模型）是瀏覽器提供給 JavaScript 操作網頁的介面。簡單來說，DOM 就是「網頁的程式化表示」，它讓 JavaScript 能夠讀取、修改、新增或刪除網頁上的任何元素。

想像一下：

  • HTML 是建築的結構（牆壁、房間、門窗）
  • CSS 是室內設計（顏色、家具、裝飾）
  • JavaScript + DOM 是智慧家居系統（可以開關燈、調整溫度、控制窗簾）

在 Figma 或 Sketch 中，你可以點選任何元素來編輯它。DOM 就是讓 JavaScript 能夠「選取並操作」網頁元素的機制。

DOM 樹狀結構

網頁的 HTML 結構會被瀏覽器轉換成樹狀結構，稱為「DOM 樹 / DOM Tree」。每個 HTML 標籤都是樹上的一個節點（node）。


<!DOCTYPE html>
<html>
  <head>
    <title>我的網站</title>
  </head>
  <body>
    <header>
      <h1>歡迎來到我的網站</h1>
    </header>
    <main>
      <p>這是內容區域</p>
    </main>
    <footer>
      <p>這是頁尾區域</p>
    </footer>
  </body>
</html>

這段 HTML 會形成這樣的樹狀結構：

html
├── head
│   └── title
└── body
    ├── header
    │   └── h1
    ├── main
    │   └── p
    └── main
        └── p

就像 Figma 的圖層面板，每個元素都有父元素、子元素、兄弟元素的關係。

為什麼要選取元素？

在開始操作網頁元素之前，我們必須先「選取」它，就像在設計軟體中，你必須先點選一個圖層才能編輯它。

常見的需求：

  • 點擊按鈕後改變文字內容
  • 滑鼠移入時改變背景顏色
  • 表單送出前驗證輸入內容
  • 顯示或隱藏某個區塊
  • 動態新增或刪除元素

JavaScript 與網頁的連結

JavaScript 透過 document 物件來存取整個網頁。document 代表整個 HTML 文件，我們可以透過它來選取任何元素。

想像 document 就是 Figma 的畫布（Canvas），所有的設計元素都在這個畫布上，我們透過 document 來找到並操作它們。

三、語法與使用方式

準備工作：建立測試網頁

首先，建立一個簡單的 HTML 檔案（index.html）：


<!DOCTYPE html>
<html lang="zh-TW">
<head>
    <meta charset="UTF-8">
    <title>DOM 練習</title>
    <style>
        body {
            font-family: Arial, sans-serif;
            padding: 20px;
            background-color: #f5f5f5;
        }
        .container {
            max-width: 800px;
            margin: 0 auto; padding: 30px;
            background: white;
            border-radius: 8px;
        }
        h1 {
            color: #2c3e50;
        }
        .highlight {
            padding: 5px;
            background-color: #ffe066;
        }
    </style>
</head>
<body>
    <div class="container">
        <h1 id="main-title">這是主標題</h1>
        <p class="intro">這是第一段文字</p>
        <p class="intro">這是第二段文字</p>
        <button id="myButton">點我！</button>
    </div>

    <script>
        // 我們的 JavaScript 程式碼會寫在這裡
        console.log("網頁載入完成！");
    </script>
</body>
</html>

在瀏覽器中開啟這個檔案，按 F12 開啟開發者工具，準備開始練習！

範例一：getElementById（透過 ID 選取）


// 選取 ID 為 "main-title" 的元素
let title = document.getElementById("main-title");

console.log(title);  // 顯示選取到的元素
console.log(title.textContent);  // 顯示元素內的文字

** 執行結果（Console）：**
<h1 id="main-title">這是主標題</h1>
這是主標題

說明： getElementById() 用於選取具有特定 ID 的元素。ID 在網頁中應該是唯一的，就像 Figma 中每個圖層都有唯一的名稱。這是最快速、最精確的選取方法。

使用時機：當你需要選取特定的單一元素時（如主標題、特定按鈕、特定區塊）。

範例二：getElementsByClassName（透過 Class 選取）


// 選取所有 class 為 "intro" 的元素
let paragraphs = document.getElementsByClassName("intro");

console.log(paragraphs);  // HTMLCollection（類似陣列）
console.log(paragraphs.length);  // 顯示找到幾個元素
console.log(paragraphs[0].textContent);  // 第一個元素的文字
console.log(paragraphs[1].textContent);  // 第二個元素的文字

** 執行結果：**
HTMLCollection(2) [p.intro, p.intro]
2
這是第一段文字
這是第二段文字

說明： getElementsByClassName() 回傳一個「集合」（HTMLCollection），包含所有符合的元素。就像在 Figma 中選取所有使用相同元件的實例。

注意： 回傳的是集合，不是陣列，但可以用索引存取（paragraphs[0]、paragraphs[1]）。

使用時機： 當你需要選取一組相同類型的元素時（如所有商品卡片、所有導覽連結）。

範例三：querySelector（CSS 選擇器，最靈活）


// 選取第一個符合條件的元素
let firstParagraph = document.querySelector(".intro");
let button = document.querySelector("#myButton");
let h1 = document.querySelector("h1");

console.log(firstParagraph.textContent);  // 這是第一段文字
console.log(button.textContent);  // 點我！
console.log(h1.textContent);  // 這是主標題

說明： querySelector() 使用 CSS 選擇器語法，非常靈活。它只回傳「第一個」符合條件的元素。

CSS 選擇器語法：

  • #id → 選取特定 ID（如 #myButton）
  • .class → 選取特定 class（如 .intro）
  • 標籤 → 選取特定標籤（如 h1、p、button）

使用時機： 最常用的方法，語法與 CSS 相同，容易記憶。

範例四：querySelectorAll（選取所有符合的元素）


// 選取所有 class 為 "intro" 的元素
let allParagraphs = document.querySelectorAll(".intro");

console.log(allParagraphs);  // NodeList（類似陣列）
console.log(allParagraphs.length);  // 2

// 使用迴圈處理每個元素
for (let i = 0; i < allParagraphs.length; i++) {
  console.log("段落 " + (i + 1) + ": " + allParagraphs[i].textContent);
}

** 執行結果：**
NodeList(2) [p.intro, p.intro]
2
段落 1: 這是第一段文字
段落 2: 這是第二段文字

說明： querySelectorAll() 回傳所有符合條件的元素（NodeList），可以搭配迴圈處理每個元素。

使用時機： 當你需要對一組元素執行相同操作時（如改變所有按鈕的顏色、隱藏所有圖片）。

範例五：進階 CSS 選擇器


// 選取 container 內的第一個 p
let firstP = document.querySelector(".container p");

// 選取所有 container 內的 p
let allPs = document.querySelectorAll(".container p");

// 選取 ID 為 myButton 的按鈕
let btn = document.querySelector("button#myButton");

console.log(firstP.textContent);
console.log(allPs.length);
console.log(btn.textContent);

說明： querySelector 支援複雜的 CSS 選擇器，可以精確定位元素。就像在 Figma 中使用圖層搜尋功能，可以找到特定結構下的元素。

範例六：完整互動範例（點擊按鈕改變標題）


// 選取元素
let title = document.getElementById("main-title");
let button = document.getElementById("myButton");

console.log("已選取標題：" + title.textContent);
console.log("已選取按鈕：" + button.textContent);

// 下一節課我們會學習如何新增點擊事件
// 現在先確認我們能成功選取元素


** 執行結果：**
已選取標題：這是主標題
已選取按鈕：點我！

四、常見錯誤與注意事項

錯誤一：在元素載入前選取

錯誤範例：


<!DOCTYPE html>
<html>
<head>
    <script>
        // 錯誤：這時候 body 還沒載入，JS 找不到 main-title 在哪裡
        let title = document.getElementById("main-title");
        console.log(title);  // null
    </script>
</head>
<body>
    <h1 id="main-title">標題</h1>
</body>
</html>

問題： JavaScript 在 <head> 中執行時，<body> 的元素還沒載入，所以選取不到，回傳 null。

正確寫法：


<!DOCTYPE html>
<html>
<head>
    <title>範例</title>
</head>
<body>
    <h1 id="main-title">標題</h1>
    
    <!-- 將 script 放在 body 結束之前 -->
    <script>
        let title = document.getElementById("main-title");
        console.log(title);  // 成功選取
    </script>
</body>
</html>

最佳實踐： 將 <script> 標籤放在 </body> 之前，確保所有 HTML 元素都已載入。

錯誤二：ID 或 Class 名稱拼錯

錯誤範例：


// HTML 中是 id="myButton"
let button = document.getElementById("mybutton");  // 小寫 b，錯誤！

console.log(button);  // null

問題： JavaScript 對大小寫敏感，myButton 和 mybutton 是不同的。

除錯技巧：

  • 檢查拼字是否正確
  • 檢查大小寫
  • 確認 HTML 中確實有這個 ID/Class
  • 在 Console 中輸入 console.log(button) 檢查是否為 null

錯誤三：忘記加 # 或 . 符號（querySelector）

錯誤範例：


// 錯誤：忘記加 # 符號
let title = document.querySelector("main-title");  // 找不到

// 正確：ID 前面要加 #
let title = document.querySelector("#main-title");  // 正確

錯誤範例：


// 錯誤：忘記加 . 符號
let paragraphs = document.querySelectorAll("intro");  // 找不到

// 正確：class 前面要加 .
let paragraphs = document.querySelectorAll(".intro");  // 正確

記憶技巧： querySelector 使用的是 CSS 選擇器語法，就像寫 CSS 一樣。

錯誤四：把 HTMLCollection 當成陣列

錯誤範例：


let paragraphs = document.getElementsByClassName("intro");

// 錯誤：HTMLCollection 沒有 forEach 方法
paragraphs.forEach(function(p) {  // 報錯！
  console.log(p.textContent);
});

正確寫法一：使用 for 迴圈


let paragraphs = document.getElementsByClassName("intro");

for (let i = 0; i < paragraphs.length; i++) {
  console.log(paragraphs[i].textContent);
}

正確寫法二：改用 querySelectorAll（推薦）


let paragraphs = document.querySelectorAll(".intro");

// querySelectorAll 回傳的 NodeList 可以使用 forEach
paragraphs.forEach(function(p) {
  console.log(p.textContent);
});

錯誤五：選取不存在的元素卻沒有檢查

錯誤範例：


let element = document.getElementById("notExist");
element.textContent = "新文字";  // 錯誤！element 是 null

**錯誤訊息：**
Uncaught TypeError: Cannot set property 'textContent' of null

正確寫法：


let element = document.getElementById("notExist");

if (element) {
  element.textContent = "新文字";
} else {
  console.log("找不到元素！");
}

五、選取方法比較表

方法 語法 回傳類型 數量 使用時機
getElementById #id 單一元素 1 選取特定 ID 的元素
getElementsByClassName .class HTMLCollection 多個 選取相同 class 的元素（舊方法）
querySelector CSS 選擇器 單一元素 1（第一個） 最靈活，推薦使用
querySelectorAll CSS 選擇器 NodeList 多個 選取多個元素，推薦使用

推薦使用：

  • 單一元素 → querySelector()
  • 多個元素 → querySelectorAll()

這兩個方法語法統一（都用 CSS 選擇器），最容易記憶和使用。

六、重點整理

  • DOM 是什麼：DOM 是瀏覽器提供的介面，讓 JavaScript 能夠操作網頁元素，就像設計軟體的圖層系統。
  • 選取元素的重要性：在操作元素之前，必須先選取它，就像在 Figma 中必須先點選圖層才能編輯。
  • 推薦方法：使用 querySelector() 選取單一元素，使用 querySelectorAll() 選取多個元素。
  • Script 位置：將 <script> 放在 </body> 之前，確保 HTML 元素已載入。
  • 檢查元素存在：選取元素後，檢查是否為 null，避免操作不存在的元素導致錯誤。
