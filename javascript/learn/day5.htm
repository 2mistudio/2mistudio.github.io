Day 5: 迴圈 (for/while)

Loops and iteration (for/while)

一、學習目標

完成本日學習後，你應該能夠：

  1. 理解迴圈的概念與應用時機
  2. 正確使用 for 迴圈處理重複性任務
  3. 正確使用 while 迴圈進行條件式重複
  4. 區分 for 與 while 的使用場景
  5. 使用 break 和 continue 控制迴圈執行

二、核心概念解釋

什麼是迴圈？

迴圈是讓程式重複執行特定程式碼區塊的控制結構。在網頁設計中，許多任務都具有重複性，例如「顯示 20 個產品卡片」、「產生導覽列的所有選單項目」、「將作品集的每張圖片都加上濾鏡」。如果沒有迴圈，我們就必須寫 20 行、30 行幾乎相同的程式碼，這不僅麻煩且難以維護。

迴圈讓程式能夠「自動化」處理重複性工作。想像你在 Figma 中需要複製 50 個相同的元件並依序編號，手動操作會很累，但如果有自動化工具（迴圈），就能一次完成。

for 迴圈

for 迴圈是最常用的迴圈結構，特別適合「已知重複次數」的情境。例如：顯示 10 個最新文章、產生 5 顆評分星星、建立 3x4 的圖片網格。

語法結構為：for (初始化; 條件; 更新) { 程式碼 }。這三個部分清楚地定義了迴圈的起點、終點與每次的變化，使程式邏輯一目了然。

while 迴圈

while 迴圈適合「重複次數未知」或「依賴動態條件」的情況。例如：持續載入更多內容直到沒有資料、不斷檢查使用者輸入直到格式正確、輪播圖片直到使用者點擊停止。

相較於 for 迴圈的結構化，while 更具彈性，但也需要更小心處理，避免產生無窮迴圈（程式永遠停不下來）。

迴圈控制：break 與 continue

break 用於立即終止迴圈，跳出整個迴圈結構。例如：在商品列表中找到目標商品後就不需要繼續找了、使用者點擊「關閉」就停止輪播。

continue 則是跳過當前迭代，直接進入下一次迭代。例如：顯示商品列表時跳過已售完的商品、只顯示已發布的文章而忽略草稿。

三、語法與使用方式

範例一：基本 for 迴圈（產生選單項目）


let menuItems = ["首頁", "關於", "作品集", "部落格", "聯絡"];

for (let i = 0; i < menuItems.length; i++) {
  console.log("選單項目 " + (i + 1) + ": " + menuItems[i]);
}

** 執行結果：**
選單項目 1: 首頁
選單項目 2: 關於
選單項目 3: 作品集
選單項目 4: 部落格
選單項目 5: 聯絡

說明： let i = 0 從第一個項目開始；i < menuItems.length 確保不超過陣列長度；i++ 每次移動到下一個項目。這個迴圈會依序處理所有選單項目。

範例二：計算購物車總價


let cartItems = [
  { name: "T恤", price: 590 },
  { name: "牛仔褲", price: 1280 },
  { name: "帽子", price: 390 },
  { name: "球鞋", price: 2200 }
];

let total = 0;

for (let i = 0; i < cartItems.length; i++) {
  total += cartItems[i].price;
  console.log("已加入：" + cartItems[i].name + " NT$ " + cartItems[i].price);
}

console.log("購物車總金額：NT$ " + total);

** 執行結果：**
已加入：T恤 NT$ 590
已加入：牛仔褲 NT$ 1280
已加入：帽子 NT$ 390
已加入：球鞋 NT$ 2200
購物車總金額：NT$ 4460

說明： 迴圈遍歷購物車中的每個商品，累加價格到總金額。這是電商網站最常見的功能之一。

範例三：產生產品網格（巢狀迴圈）


// 產生 3 列 x 4 欄的產品網格
for (let row = 1; row <= 3; row++) {
  let rowText = "第 " + row + " 列：";
  
  for (let col = 1; col <= 4; col++) {
    rowText += " [產品" + ((row-1)*4 + col) + "]";
  }
  
  console.log(rowText);
}

** 執行結果：**
第 1 列： [產品1] [產品2] [產品3] [產品4]
第 2 列： [產品5] [產品6] [產品7] [產品8]
第 3 列： [產品9] [產品10] [產品11] [產品12]

說明： 巢狀迴圈（迴圈內包含迴圈）用於建立二維結構。外層控制列數，內層控制每列的欄數。這在建立圖片網格、表格、棋盤等場景中非常實用。

範例四：while 迴圈（載入更多內容）


let loadedItems = 0;
let totalItems = 5;

console.log("開始載入內容...");

while (loadedItems < totalItems) {
  loadedItems++;
  console.log("已載入項目 " + loadedItems + "/" + totalItems);
}

console.log("所有內容載入完成！");

** 執行結果：**
開始載入內容...
已載入項目 1/5
已載入項目 2/5
已載入項目 3/5
已載入項目 4/5
已載入項目 5/5
所有內容載入完成！

說明： while 迴圈會先檢查條件，條件為 true 才執行。這個範例模擬「載入更多」的功能，持續載入直到達到總數量。

範例五：使用 break 提前結束（搜尋功能）


let products = ["筆電", "滑鼠", "鍵盤", "螢幕", "耳機"];
let searchTarget = "鍵盤";
let found = false;

for (let i = 0; i < products.length; i++) {
  console.log("正在檢查：" + products[i]);
  
  if (products[i] === searchTarget) {
    console.log("✓ 找到了！" + searchTarget + " 在第 " + (i + 1) + " 個位置");
    found = true;
    break;  // 找到後就不需要繼續找了
  }
}

if (!found) {
  console.log("✗ 找不到 " + searchTarget);
}

** 執行結果：**
正在檢查：筆電
正在檢查：滑鼠
正在檢查：鍵盤
✓ 找到了！鍵盤在第 3 個位置

說明： 當 i 等於 5 時，執行 break，整個迴圈立即終止，後續的 6、7、8、9、10 都不會被執行。

範例六：使用 continue 跳過迭代


for (let i = 1; i <= 10; i++) {
  if (i % 2 === 0) {
    continue;  // 跳過偶數
  }
  console.log(i);
}

**執行結果：**
1
3
5
7
9

說明： 當 i 是偶數時（i % 2 === 0），執行 continue，跳過當次迭代的剩餘程式碼，直接進入下一次迭代。因此只有奇數會被印出。

四、常見錯誤與注意事項

錯誤一：無窮迴圈

錯誤範例：


let i = 1;

while (i <= 5) {
  console.log(i);
  // 忘記更新 i，i 永遠是 1，條件永遠成立
}

問題： 沒有更新迴圈變數，條件永遠為 true，迴圈無法停止，程式會當掉或瀏覽器失去回應。

正確寫法：


let i = 1;

while (i <= 5) {
  console.log(i);
  i++;  // 必須更新變數
}

錯誤二：off-by-one 錯誤

錯誤範例：


// 想要執行 10 次，但實際只執行 9 次
for (let i = 1; i < 10; i++) {
  console.log(i);
}

問題： 條件是 i < 10，當 i = 10 時就停止，所以只執行到 i = 9。如果想執行 10 次，應該用 i <= 10 或 i < 11。

正確寫法：


for (let i = 1; i <= 10; i++) {
  console.log(i);
}

錯誤三：在迴圈內宣告重複變數

不良範例：


for (let i = 0; i < 3; i++) {
  let sum = 0;  // 每次迴圈都重新宣告並初始化
  sum += i;
  console.log(sum);  // 永遠顯示當前的 i 值
}

** 執行結果：**
0
1
2

問題： 如果希望 sum 累加所有的 i，應該將 sum 宣告在迴圈外。


let sum = 0;

for (let i = 0; i < 3; i++) {
  sum += i;
}

console.log(sum);  // 3

錯誤四：巢狀迴圈變數重複

錯誤範例：


for (let i = 1; i <= 3; i++) {
  for (let i = 1; i <= 3; i++) {  // 內外層都用 i
    console.log(i);
  }
}

問題： 內外層迴圈使用相同的變數名稱會造成混亂，內層會覆蓋外層的值。

正確寫法：


for (let i = 1; i <= 3; i++) {
  for (let j = 1; j <= 3; j++) {  // 使用不同變數名
    console.log("i=" + i + ", j=" + j);
  }
}

錯誤五：誤用 break 和 continue

錯誤範例：


for (let i = 1; i <= 5; i++) {
  if (i === 3) {
    break;  // 想跳過 3，但誤用 break
  }
  console.log(i);
}

** 執行結果：**
1
2

問題： break 會終止整個迴圈，而非只跳過當次。如果只想跳過某個值，應該用 continue。

正確寫法：


for (let i = 1; i <= 5; i++) {
  if (i === 3) {
    continue;  // 跳過 3，繼續執行
  }
  console.log(i);
}

** 執行結果：**
1
2
4
5

五、重點整理

  • for 迴圈：適合已知次數的重複任務，結構清晰，包含初始化、條件、更新三部分。
  • while 迴圈：適合未知次數或依賴動態條件的情境，更有彈性但需注意避免無窮迴圈。
  • 避免無窮迴圈：務必確保迴圈條件最終會變成 false，while 迴圈內必須更新影響條件的變數。
  • break 與 continue：break 終止整個迴圈，continue 跳過當次迭代繼續下一次。
  • 巢狀迴圈：內外層使用不同的變數名稱，注意執行次數為外層次數 × 內層次數。
