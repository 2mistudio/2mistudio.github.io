Day 6: Switch、三元運算子與函式入門

Switch, Ternary Operator and Function

一、學習目標

完成本日學習後，你應該能夠：

  1. 理解 switch 語句的結構與使用時機
  2. 使用三元運算子簡化簡單的條件判斷
  3. 理解函式的概念與基本用途
  4. 能夠定義並呼叫簡單的函式
  5. 理解參數與回傳值的基本概念

二、核心概念解釋

Switch 與三元運算子

switch 是一種多分支選擇結構，就像設計軟體中的「圖層樣式選單」，根據你選擇的不同選項（陰影、描邊、漸層...），套用不同的效果。當需要根據某個變數的不同值執行不同的程式碼時，switch 比多個 if/else if 更簡潔易讀。

三元運算子（ternary operator）是一種簡化的條件表達式，語法為：條件 ? 值1 : 值2。就像 Photoshop 的圖層開關，如果「開」就顯示，如果「關」就隱藏，只有兩種狀態。

什麼是函式？

函式（Function）是一段可重複使用的程式碼區塊。就像 Photoshop 的「Action / 動作」或 Figma 的（Component / 元件）。當你需要重複執行相同的操作時，與其每次都重新手動做一遍，不如把這些步驟「打包」存起來，需要時一鍵執行。

其實，從一開始，我們就一直在使用函式，例如 console.log()、prompt()、Math.random() 等，這些都是 JavaScript 內建的函式。現在我們要學習如何建立自己的函式。

為什麼需要函式？

程式碼重複使用：就像在 Figma 中建立 Component，修改一次就能更新所有實例。相同的邏輯只需寫一次，可以在不同地方重複呼叫。

提高可讀性：將複雜的邏輯封裝成有意義的函式名稱，程式碼更易理解。例如 changeButtonColor() 比一長串程式碼更清楚表達意圖。

易於維護：當需要調整某個功能時，只需改一個地方，就像修改設計系統中的主色票，所有使用該顏色的地方都會自動更新。

模組化思維：將大任務拆解成小任務，每個函式負責一件事。就像設計流程：先做線框圖、再做視覺稿、最後做原型，每個階段專注於一個目標。

函式的基本結構

函式由三個部分組成：

  • 函式名稱：有意義的名稱，描述函式的功能（如 showMessage、hideMenu）
  • 參數（可選）：函式需要的輸入資料（如要顯示的文字、要改變的顏色）
  • 回傳值（可選）：函式執行後輸出的結果

定義函式不會立即執行，必須「呼叫」函式才會執行其中的程式碼，就像建立 Photoshop Action 後，還要手動播放才會執行。

三、語法與使用方式

範例一：基本 switch 語句（UI 主題切換）


let theme = "dark";
let backgroundColor;

switch (theme) {
  case "light":
    backgroundColor = "#FFFFFF";
    break;
  case "dark":
    backgroundColor = "#1A1A1A";
    break;
  case "blue":
    backgroundColor = "#0066CC";
    break;
  default:
    backgroundColor = "#FFFFFF";
}

console.log("背景色：" + backgroundColor);  // 背景色：#1A1A1A

說明： 就像設計軟體的主題切換器，根據使用者選擇的主題（淺色、深色、藍色），套用對應的背景顏色。

範例二：三元運算子（顯示/隱藏元素）


let isMenuOpen = true;
let menuStatus = isMenuOpen ? "顯示" : "隱藏";

console.log("選單狀態：" + menuStatus);  // 選單狀態：顯示

說明： 這就像設計軟體中的圖層可見性開關，開啟就顯示，關閉就隱藏，只有兩種狀態。

範例三：定義與呼叫函式（顯示訊息）


// 定義函式
function showWelcomeMessage() {
  console.log("歡迎來到我的網站！");
}

// 呼叫函式
showWelcomeMessage();  // 歡迎來到我的網站！
showWelcomeMessage();  // 歡迎來到我的網站！

說明： 就像建立一個可重複使用的「歡迎訊息」元件，需要時隨時呼叫。可以多次使用同一個函式。

範例四：帶參數的函式（客製化訊息）


// 定義帶參數的函式
function greetUser(userName) {
  console.log("嗨，" + userName + "！歡迎回來！");
}

// 呼叫函式並傳入不同的使用者名稱
greetUser("小明");  // 嗨，小明！歡迎回來！
greetUser("Amy");   // 嗨，Amy！歡迎回來！

說明： 參數讓函式更靈活，就像 Figma 的 Component Variants，可以根據不同的輸入產生不同的結果。userName 是參數，"小明"、"Amy" 是傳入的值。

範例五：多個參數的函式（按鈕樣式設定）


function setButtonStyle(textColor, bgColor) {
  console.log("按鈕文字顏色：" + textColor);
  console.log("按鈕背景顏色：" + bgColor);
}

setButtonStyle("#FFFFFF", "#0066CC");  // 白色文字，藍色背景
setButtonStyle("#000000", "#FFD700");  // 黑色文字，金色背景

說明： 函式可以接收多個參數，就像設定設計元件的多個屬性。參數的順序很重要，第一個參數對應文字顏色，第二個對應背景顏色。

範例六：有回傳值的函式（計算折扣價）


function calculateDiscountPrice(originalPrice, discountPercent) {
  let discount = originalPrice * (discountPercent / 100);
  let finalPrice = originalPrice - discount;
  return finalPrice;  // 回傳計算結果
}

let price1 = calculateDiscountPrice(1000, 20);  // 原價 1000，打 8 折
let price2 = calculateDiscountPrice(500, 10);   // 原價 500，打 9 折

console.log("折扣後價格：" + price1);  // 折扣後價格：800
console.log("折扣後價格：" + price2);  // 折扣後價格：450

說明： return 用於回傳函式的結果。回傳值可以存入變數中，或直接使用。這就像電商網站的價格計算器，輸入原價和折扣，回傳最終價格。

範例七：實用函式範例（檢查 Email 格式）


// 判斷是否為有效的 Email 格式（簡化版）
function isValidEmail(email) {
  if (email.includes("@") && email.includes(".")) {
    return true;
  } else {
    return false;
  }
}

console.log(isValidEmail("user@example.com"));  // true
console.log(isValidEmail("invalidemail"));      // false

說明： 檢查 Email 格式（表單驗證常用）。

四、常見錯誤與注意事項

錯誤一：忘記呼叫函式

錯誤範例：


function showAlert() {
  console.log("這是一個警告訊息！");
}

showAlert;  // 錯誤：忘記加小括號

問題： 只寫函式名稱不會執行函式，必須加上小括號 () 才是呼叫。就像在 Figma 中，你建立了 Component 但沒有實際拖曳使用。

正確寫法：


showAlert();  // 正確：加上小括號

錯誤二：參數數量不匹配

容易混淆的範例：


function displayNotification(title, message) {
  console.log(title + ": " + message);
}

displayNotification("提醒");  // 提醒: undefined

說明： 如果傳入的值少於參數數量，缺少的參數會是 undefined。雖然不會報錯，但結果可能不完整。就像設計組件缺少某個必要的屬性。

正確寫法：


displayNotification("提醒", "您有新訊息");  // 提醒: 您有新訊息

錯誤三：忘記 return

錯誤範例：


function getButtonText(isLoggedIn) {
  if (isLoggedIn) {
    "登出";  // 忘記 return
  } else {
    "登入";  // 忘記 return
  }
}

let buttonText = getButtonText(true);
console.log(buttonText);  // undefined

問題： 沒有 return 的函式會回傳 undefined。如果需要使用函式的結果，必須加上 return。

正確寫法：


function getButtonText(isLoggedIn) {
  if (isLoggedIn) {
    return "登出";
  } else {
    return "登入";
  }
}

let buttonText = getButtonText(true);
console.log(buttonText);  // 登出

錯誤四：return 後的程式碼不會執行

錯誤範例：


function showMessage() {
  return "訊息已顯示";
  console.log("這行永遠不會執行");  // 死代碼
}

showMessage();

說明： 執行到 return 後，函式立即結束，後面的程式碼都不會執行。就像設計流程中，如果案子在「線框圖」階段就被客戶否決，後面的「視覺稿」和「原型」階段就不會進行。

錯誤五：switch 忘記 break（顏色設定穿透）

錯誤範例：


let colorMode = "primary";

switch (colorMode) {
  case "primary":
    console.log("主色：藍色");
    // 忘記 break
  case "secondary":
    console.log("次要色：灰色");
    // 忘記 break
  case "accent":
    console.log("強調色：橘色");
    break;
}

**執行結果：**
主色：藍色
次要色：灰色
強調色：橘色

問題： 匹配到 "primary" 後，因為沒有 break，會繼續執行下面所有的 case，直到遇到 break。


switch (colorMode) {
  case "primary":
    console.log("主色：藍色");
    break;
  case "secondary":
    console.log("次要色：灰色");
    break;
  case "accent":
    console.log("強調色：橘色");
    break;
}

五、重點整理

  • switch 語句：適合處理多個選項的情況（如主題切換、狀態選擇），每個 case 後記得加 break。
  • 三元運算子：語法為 條件 ? 值1 : 值2，適合簡單的開關狀態（顯示/隱藏、啟用/停用）。
  • 函式定義：使用 function 函式名稱() { 程式碼 } 定義函式，可以重複使用相同邏輯。
  • 參數與引數：參數是函式定義時的變數，引數是呼叫時傳入的實際值。就像設計組件的可變屬性。
  • 回傳值：使用 return 回傳函式結果，執行到 return 後函式立即結束。
